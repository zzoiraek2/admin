# Admin 설계정책서 개발 지침

이 프로젝트의 메뉴별 설계는 개발 착수 시 기준 문서로 사용합니다. 개발 요청, 이슈, PR, 작업 브랜치에서는 메뉴명을 길게 풀어 쓰기보다 화면에 표시되는 개발 참조 코드를 함께 적습니다.

## 지시 방식

권장 지시문:

```text
ADM-MEM-001 설계대로 회원 기본조회 화면을 개발해.
ADM-WAL-004 설계 기준으로 출금 승인/보류 화면의 목록, 상세, 권한, 감사로그를 구현해.
ADM-RISK-002 정책에서 벗어나는 부분은 변경 사유를 남기고 대체 정책을 제안해.
```

메뉴 상세 화면의 `지침 복사` 버튼을 누르면 해당 메뉴의 표준 개발 지시문을 복사할 수 있습니다.

## 코드 체계

메뉴 코드는 상단 업무 도메인과 CSV 순서를 기준으로 자동 부여합니다.

| 상단 메뉴 | Prefix | 예시 |
| --- | --- | --- |
| 대시보드 | `ADM-DASH` | `ADM-DASH-001` |
| 회원·고객확인 | `ADM-MEM` | `ADM-MEM-001` |
| 거래·마켓 관리 | `ADM-MKT` | `ADM-MKT-001` |
| 입출금·지갑 | `ADM-WAL` | `ADM-WAL-001` |
| 준법·FDS 관리 | `ADM-RISK` | `ADM-RISK-001` |
| 거래지원·상품 | `ADM-LIST` | `ADM-LIST-001` |
| 정산·회계 | `ADM-SETTLE` | `ADM-SETTLE-001` |
| 고객지원·서비스 | `ADM-CS` | `ADM-CS-001` |
| 보안·관리자통제 | `ADM-SEC` | `ADM-SEC-001` |

## 설계 기준 범위

개발자가 메뉴 코드를 참조하면 아래 항목을 함께 따라야 합니다.

- 화면 목적
- 업무 설명
- 탭 구성 또는 탭 제안
- 설계 방식 제안
- 주요 액션
- 권한과 감사로그 기준
- 정책 체크리스트
- 구 어드민 매핑이 있는 경우 기존 업무 흐름

## 구현 원칙

설계서와 구현이 다를 때는 코드에 임의로 맞추지 말고 아래 내용을 작업 기록에 남깁니다.

- 대상 메뉴 코드
- 달라지는 설계 항목
- 변경 사유
- 대체 정책
- 영향 범위
- 승인 또는 확인이 필요한 담당자

## 코드 변경 규칙

CSV의 상단 메뉴나 행 순서를 바꾸면 메뉴 코드가 달라질 수 있습니다. 이미 개발이 시작된 메뉴는 코드 변경 영향을 확인한 뒤 진행합니다.

운영 중인 개발 이슈나 PR에서 참조 중인 코드는 임의로 재배치하지 않습니다. 메뉴를 추가해야 한다면 같은 도메인 마지막에 추가하는 방식을 우선합니다.
