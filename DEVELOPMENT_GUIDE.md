# Admin 설계정책서 개발 지침

이 프로젝트는 메뉴별 설계정책을 개발 착수 시 기준 문서로 사용하기 위한 작업공간입니다. 개발 요청, 이슈, PR, 작업 브랜치에서는 메뉴명을 길게 풀어 쓰기보다 메뉴 코드와 MD 문서 경로를 함께 적는 방식을 권장합니다.

## 지시 방식

권장 지시문:

```text
ADM-MEM-001 docs/menus/ADM-MEM-001.md 설계대로 회원 기본조회 화면을 개발해.
ADM-WAL-004 설계 기준으로 출금 승인/보류 화면의 목록, 상세, 권한, 감사로그를 구현해.
ADM-RISK-004 설계와 다르게 구현해야 하는 부분은 변경 사유와 대체 정책을 먼저 정리해.
```

## 문서 원천

- 메뉴별 원천 문서: `docs/menus/{MENU_CODE}.md`
- 도메인 보강 문서: `docs/domains/{DOMAIN}.md`
- 메뉴 상태 원천: 각 메뉴 MD 파일의 `- 상태:` 값

웹 UI에서 상태를 변경하면 로컬 관리 서버가 해당 MD 파일의 `- 상태:` 줄을 수정합니다. 상태 변경은 Git diff로 남기고 커밋/푸시해서 공유합니다.

## 메뉴 코드 체계

메뉴 코드는 MD 파일명과 문서 내부 `메뉴 코드`를 기준으로 고정합니다. 이미 개발이 시작된 메뉴의 코드는 임의로 바꾸지 않습니다.

| 업무 도메인 | Prefix | 예시 |
| --- | --- | --- |
| 대시보드 | `ADM-DASH` | `ADM-DASH-001` |
| 회원·고객확인 | `ADM-MEM` | `ADM-MEM-001` |
| 거래·마켓 관리 | `ADM-MKT` | `ADM-MKT-001` |
| 입출금·지갑 | `ADM-WAL` | `ADM-WAL-001` |
| 준법·FDS 관리 | `ADM-RISK` | `ADM-RISK-001` |
| 거래지원·상품 | `ADM-LIST` | `ADM-LIST-001` |
| 정산·회계 | `ADM-SETTLE` | `ADM-SETTLE-001` |
| 고객지원·서비스 | `ADM-CS` | `ADM-CS-001` |
| 보안·관리자통제 | `ADM-SEC` | `ADM-SEC-001` |

## 설계 기준 범위

개발자가 메뉴 코드를 참조하면 아래 항목을 함께 따라야 합니다.

- 화면 목적
- 업무 설명
- 탭 구성 또는 화면 구성 제안
- 설계 방식 제안
- 주요 액션
- 권한과 감사로그 기준
- 정책 체크리스트
- 구 어드민 매핑이 있는 경우 기존 업무 흐름

## 탭 설계 원칙

탭은 데이터 필드 묶음이 아니라 운영자가 같은 대상을 다른 업무 관점으로 전환해서 보는 화면 구조로 정의합니다.

- `기본정보`, `연락처`, `가입정보`, `상태`, `태그`처럼 컬럼 그룹을 탭으로 나누지 않는다.
- 필드 그룹은 탭이 아니라 요약 카드, 상세 테이블, 섹션, 접힘 영역으로 배치한다.
- 조회 화면의 탭은 `요약`, `거래·입출금`, `KYC·심사`, `보안·접속`, `제한·리스크`, `처리이력`, `감사로그`처럼 업무 관점으로 나눈다.
- 처리 화면의 탭은 `요청`, `검토`, `승인·반려`, `적용 결과`, `감사로그`처럼 프로세스 관점으로 나눈다.
- 모니터링 화면의 탭은 `실시간`, `기간별`, `대상별`, `알림·이벤트`, `처리현황`처럼 분석 관점으로 나눈다.
- 설정 화면의 탭은 `정책 요약`, `대상별 설정`, `검증·적용`, `변경 이력`처럼 운영 행위와 영향 범위 중심으로 나눈다.

## 구현 원칙

설계서 구현 시 아래 항목은 작업 기록에 남깁니다.

- 대상 메뉴 코드
- 따라간 MD 문서 경로
- 설계와 달라지는 항목
- 변경 사유
- 대체 정책
- 영향 범위
- 승인 또는 확인이 필요한 항목

## ADM-RISK-001/002 차단 등록과 해제

`ADM-RISK-001`은 차단을 등록하는 화면이고, `ADM-RISK-002`는 이미 차단된 대상자의 차단을 해제하는 화면입니다. 두 화면 모두 단순 등록/해제 버튼이 아니라 승인 기반 고위험 업무로 구현합니다.

`ADM-RISK-001 차단 등록` 개발 기준:

- 고객 대상자를 지정한 뒤 차단 등록
- ADM-RISK-003 차단 사유코드 선택
- 회원, 거래, 입금, 출금, 입고, 출고 등 차단 범위 선택
- FDS, KYT, KYC 미흡, 제재국가, 트래블룰 미준수 등 자동 등록 이벤트 연계
- 수동 등록과 자동 등록 구분
- 등록 요청, 승인, 반려, 적용 결과, 전후 상태 감사로그 기록

`ADM-RISK-002 차단 해제` 개발 기준:

- FDS 차단 고객, 수동 차단 등록 대상자, KYC·제재·트래블룰 사유 제한 고객의 해제 요청 처리
- 원 차단 사유, 등록자, 승인자, 차단 범위, 관련 Case 확인
- 해제 사유, 증빙, 조치 결과 입력
- 전체 해제와 부분 해제 구분
- 승인 완료 후 해제 적용
- 해제 요청, 승인, 반려, 보류, 적용 결과 감사로그 기록

## ADM-RISK-003 차단 사유코드 관리

`ADM-RISK-003`은 입출금 차단 보고 기준과 사유별 산출 기준을 관리하는 정책 화면입니다.

개발 시 아래 기준을 포함합니다.

- 차단 사유 번호 1~13 관리
- 코인거래소용 가상자산 입출고 사유 관리
- 원화거래소용 예치금 입출금 사유 관리
- 사유별 건 수 산출 기준 관리
- 산출 기준 값: 사건발생일, 가상자산종목수, 이용자수
- 입금, 출금, 입고, 출고 차단 방향 관리
- 사유코드 변경 이력, 사용 여부, 승인 상태, 감사로그 관리

## ADM-RISK-004 차단 관리대장

`ADM-RISK-004`는 차단 사유별 일계·월계 집계와 최초 보고 근거를 관리하는 화면입니다. 원천 로그를 단순 합산하는 화면이 아니라 `ADM-RISK-003`의 산출 기준에 따라 숫자, 날짜, 전체, 종목, 금액을 다르게 표시해야 합니다.

개발 기준:

- 탭은 `차단 관리대장`과 `원천 로그` 2개로 구성
- `차단 관리대장` 탭은 기간 검색, 전월 기준 조회, 일계/월계 집계, 사유코드별 집계표를 제공
- `원천 로그` 탭은 건 단위 누적현황을 조회하고 관리대장 집계값의 산출 근거로 사용
- 차단 등록, 자동 차단 로그, 사건 기록성 등록, 수동 보정은 원천으로 관리
- 일계 집계와 월계 집계 제공
- `이용자수` 기준은 회원 중복 제거 후 숫자 합계
- `가상자산종목수` 기준은 종목 수 또는 종목 목록, 전체 대상이면 `전체`
- `사건발생일` 기준은 날짜 목록 또는 발생일 수
- 가상자산 입고/출고/입출고 차단과 예치금 입금/출금/입출금 차단을 구분
- 최초 보고일자, 최초보고 문서번호, 차단사유 근거 관리
- 수동 보정은 보정 전후 값, 보정 사유, 승인자, 증빙을 기록

## ADM-RISK-013 KYT VASP 차단 관리

`ADM-RISK-013`은 KYT VASP 차단 관리 화면입니다. 미신고 VASP 관리는 별도 메뉴로 두지 않고 이 메뉴에서 통합 관리합니다.

관리 항목:

- No
- VASP명
- Cluster Name
- Site
- Status
- 내부 비고

이 화면은 입출금 차단 행위를 직접 생성하는 화면이 아니라 KYT 판단과 차단 정책에서 참조할 VASP 식별 목록을 관리하는 기준정보 화면입니다.
