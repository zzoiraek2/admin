# ADM-MEM-006 로그인/보안상태

## 기본 정보

- 메뉴 코드: ADM-MEM-006
- 상단 메뉴: 회원·고객확인
- 좌측 1메뉴: 회원 인증관리
- 좌측 2메뉴: 로그인/보안상태
- 상태: 정의됨
- 구 어드민 매핑: 회원관리 회원관리 > 회원관리 > 회원 로그인 정보 

## 화면 목적

고객 로그인·보안수단 상태 및 OTP 해제 이력 확인

## 업무 설명

최근 로그인, 실패 이력, 접속 IP, 기기정보, OTP/MFA 상태, OTP 해제 이력, 전자서명 검증, 위험 판정, 고객 알림


## 결재 프로세스

- 인증수단 상태: 인증수단 초기화나 관리자 수동 해제는 승인 대상이다. 고객 전자서명 기반 정상 해제는 자동 처리 후 이력만 남긴다.
- OTP 해제 이력: 전자서명 성공과 위험조건 없음은 자동 해제로 결재상신이 없다. 전자서명 실패, 고위험, 수동 보정, 예외 해제는 승인 대상이다.
- 이상접속·제한: 자동 제한은 정책 기반으로 반영된다. 수동 해제, 예외 인정, 정책 변경은 승인 대상이다.

## 탭 구성

- 보안 요약
- 접속·기기 이력
- 인증수단 상태
- OTP 해제 이력
- 이상접속·제한
- 감사로그


## 탭별 상세 설계

### 보안 요약

- 업무 목적: 회원의 로그인 가능 상태, 잠금 여부, 보안등급, 인증수단 보유 상태를 한 화면에서 판단하는 탭이다.
- 검색 조건: 회원번호, UID, 이름, 이메일, 휴대폰, 계정상태, 잠금상태, 보안등급
- 목록 컬럼: 회원번호, 계정상태, 로그인 잠금, 보안등급, OTP/MFA 상태, 최근 접속, 이상징후, 담당자
- 상세보기 정보: 계정 보안 요약, 로그인 가능 여부, 인증수단 구성, 최근 위험 이벤트, 상담/처리 링크
- 주요 기능: 보안 상태 조회, 관련 접속 이력 이동, 상담 메모 참고, 이상접속 Case 이동
- 결재/승인: 요약 조회는 결재상신 대상이 아니다. 변경성 조치는 인증수단 상태, OTP 해제 이력, 이상접속·제한 탭에서 판단한다.
- 연계 메뉴: 회원 기본조회, KYC 심사 대기, FDS Case, 차단 등록/해제, 상담 이력, 감사로그
- 권한/감사: 보안 상태 조회, 인증수단 상태 변경, OTP 해제 이력, 전자서명 검증, 제한/해제와 다운로드 사유를 감사로그로 남긴다.

### 접속·기기 이력

- 업무 목적: 최근 로그인, 실패 로그인, 접속 IP, 기기 변경 이력을 조회해 계정도용 위험을 확인하는 탭이다.
- 검색 조건: 기간, 회원번호, IP, 기기ID, 로그인 결과, 실패사유, 국가, 위험판정
- 목록 컬럼: 접속일시, IP/국가, 기기ID, 로그인 결과, 실패사유, 위험판정, 알림 여부
- 상세보기 정보: 접속 상세, 실패 원인, 기기/브라우저 정보, 국가/IP 평판, 관련 FDS Case
- 주요 기능: 접속 이력 조회, 기기 이력 비교, 이상접속 Case 연결, 로그 다운로드
- 결재/승인: 조회와 모니터링은 결재상신 대상이 아니다. 로그 다운로드는 별도 권한과 사유 입력을 요구한다.
- 연계 메뉴: 회원 기본조회, KYC 심사 대기, FDS Case, 차단 등록/해제, 상담 이력, 감사로그
- 권한/감사: 보안 상태 조회, 인증수단 상태 변경, OTP 해제 이력, 전자서명 검증, 제한/해제와 다운로드 사유를 감사로그로 남긴다.

### 인증수단 상태

- 업무 목적: OTP, MFA, 생체/간편인증, 전자서명 수단의 현재 상태와 변경 이력을 조회하는 탭이다.
- 검색 조건: 회원번호, 인증수단, 등록상태, 변경일, 잠금여부, 해제여부
- 목록 컬럼: 인증수단, 등록상태, 최근 변경일, 잠금상태, 실패횟수, 해제 이력, 처리결과
- 상세보기 정보: 인증수단별 등록/해제/잠금 이력, 실패횟수, 전자서명 연계 상태, 고객 알림 이력
- 주요 기능: 인증수단 상태 조회, 관련 OTP 해제 이력 이동, 잠금/초기화 예외 Case 생성
- 결재/승인: 인증수단 초기화나 관리자 수동 해제는 승인 대상이다. 고객 전자서명 기반 정상 해제는 자동 처리 후 이력만 남긴다.
- 연계 메뉴: 회원 기본조회, KYC 심사 대기, FDS Case, 차단 등록/해제, 상담 이력, 감사로그
- 권한/감사: 보안 상태 조회, 인증수단 상태 변경, OTP 해제 이력, 전자서명 검증, 제한/해제와 다운로드 사유를 감사로그로 남긴다.

### OTP 해제 이력

- 업무 목적: 별도 메뉴로 관리하지 않고 고객이 프론트에서 본인확인/전자서명으로 OTP 해제를 요청한 이력과 처리 결과를 조회하는 탭이다.
- 검색 조건: 기간, 회원번호, 요청경로, 전자서명 검증결과, 위험판정, 처리결과, 해제시각
- 목록 컬럼: 요청일시, 요청경로, 전자서명 결과, 위험판정, 처리결과, 해제시각, 출금지연 여부, 알림 발송
- 상세보기 정보: 요청 원문, 전자서명 검증 요약, 위험조건, 자동/예외 처리 사유, 해제 전후 상태, 고객 알림, 감사로그
- 주요 기능: 이력 조회, 실패/예외 Case 이동, 관련 증빙/상담 연결, 출금지연 상태 확인
- 결재/승인: 전자서명 성공과 위험조건 없음은 자동 해제로 결재상신이 없다. 전자서명 실패, 고위험, 수동 보정, 예외 해제는 승인 대상이다.
- 연계 메뉴: 회원 기본조회, KYC 심사 대기, FDS Case, 차단 등록/해제, 상담 이력, 감사로그
- 권한/감사: 보안 상태 조회, 인증수단 상태 변경, OTP 해제 이력, 전자서명 검증, 제한/해제와 다운로드 사유를 감사로그로 남긴다.

### 이상접속·제한

- 업무 목적: 계정도용 의심, 제재국가 IP, 반복 실패 등 보안 위험에 따른 잠금·제한 상태를 모니터링하는 탭이다.
- 검색 조건: 회원번호, 위험유형, 제한상태, 발생일시, 원천시스템, 해제상태
- 목록 컬럼: 발생일시, 위험유형, 제한범위, 원천시스템, 처리상태, 담당자, 관련 Case
- 상세보기 정보: 위험 근거, 제한 전후 상태, 자동 제한 정책, 해제 조건, 관련 FDS/KYC Case
- 주요 기능: 제한 상태 조회, 관련 차단/해제 Case 이동, 고객 안내 이력 확인, 예외 검토 요청
- 결재/승인: 자동 제한은 정책 기반으로 반영된다. 수동 해제, 예외 인정, 정책 변경은 승인 대상이다.
- 연계 메뉴: 회원 기본조회, KYC 심사 대기, FDS Case, 차단 등록/해제, 상담 이력, 감사로그
- 권한/감사: 보안 상태 조회, 인증수단 상태 변경, OTP 해제 이력, 전자서명 검증, 제한/해제와 다운로드 사유를 감사로그로 남긴다.

### 감사로그

- 업무 목적: 로그인/보안 상태 조회, 인증수단 변경, OTP 해제, 제한/해제 이력을 감사 기준으로 추적하는 탭이다.
- 검색 조건: 기간, 관리자ID, 작업유형, 회원번호, IP, 결과, 승인상태
- 목록 컬럼: 작업일시, 관리자, 작업유형, 대상 회원, 변경 전후 값, 사유, IP, 결과
- 상세보기 정보: 조회 사유, 마스킹 해제 사유, 인증수단 변경 전후 값, 승인 의견, 전자서명 검증 결과
- 주요 기능: 감사 로그 조회, 변경 전후 비교, 관련 승인 건 이동, 로그 다운로드
- 결재/승인: 감사로그는 수정할 수 없으며 다운로드에는 별도 권한과 사유 입력을 요구한다.
- 연계 메뉴: 회원 기본조회, KYC 심사 대기, FDS Case, 차단 등록/해제, 상담 이력, 감사로그
- 권한/감사: 보안 상태 조회, 인증수단 상태 변경, OTP 해제 이력, 전자서명 검증, 제한/해제와 다운로드 사유를 감사로그로 남긴다.

## 설계 방식 제안

- 목록, 상세, 처리 이력, 변경 이력을 기준 구조로 둔다.
- 조회 권한과 처리 권한을 분리한다.
- 변경성 액션은 사유 입력과 감사로그를 필수로 둔다.
- 관련 회원, 거래, 입출금, Case, 보고서 화면으로 이동할 수 있게 연결한다.
