## 화면 목적

지갑 잔고와 정산 원장 대사

## 업무 설명

지갑별 온체인 잔고, 내부원장, 차이, 보정

## 탭 구성

- 정산 요약
- 기간별 집계
- 대사·차이
- 보정·승인
- 다운로드·감사

## 설계 방식 제안

- 목록, 상세, 처리 이력, 변경 이력을 기준 구조로 둔다.
- 조회 권한과 처리 권한을 분리한다.
- 변경성 액션은 사유 입력과 감사로그를 필수로 둔다.
- 관련 회원, 거래, 입출금, Case, 보고서 화면으로 이동할 수 있게 연결한다.

## 개발 지침

ADM-SETTLE-010 화면은 이 문서의 화면 목적, 탭 구성, 설계 방식, 권한/감사 기준을 기준으로 개발한다.
설계와 다르게 구현해야 하는 항목은 변경 사유와 대체 정책을 작업 기록에 남긴다.
