# ADM-SEC-010 중요작업 이력

## 기본 정보

- 메뉴 코드: ADM-SEC-010
- 상단 메뉴: 보안·관리자통제
- 좌측 1메뉴: 관리자 작업로그
- 좌측 2메뉴: 중요작업 이력
- 상태: 정의됨
- 구 어드민 매핑: 신규 정책 정의

## 화면 목적

## 업무 설명

## 탭 구성

## 탭별 상세 설계

### 관리자 요약

- 업무 목적: 중요작업 이력의 관리자 요약 탭은 보안 관리자, 시스템 관리자, 감사 담당자, 승인권자가 관리자 권한, 접근 통제, 개인정보 조회와 시스템 설정 관련 정보를 확인하거나 처리할 때 대상의 현재 상태와 미처리 위험을 빠르게 판단하는 탭이다.
- 검색 조건: 관리자ID, 역할, 권한, IP, 기간, 작업유형, 승인상태, 이상징후, 기간, 상태, 담당자, 위험등급, 미처리 여부
- 목록 컬럼: 관리자, 역할/권한, 대상 메뉴, 작업유형, 사유, IP/기기, 승인상태, 작업시각, 상태, 핵심 수치, 최근 변경, 담당자, 바로가기
- 상세보기 정보: 권한 범위, 접근 이력, 작업 전후 값, 조회 사유, 마스킹 해제 여부, 승인/반려 이력, 요약 카드, 핵심 지표 산출 기준, 관련 최근 이벤트, 다음 처리 링크
- 주요 기능: 권한 부여/회수 상신, 세션 종료, 설정 변경, 로그 다운로드, 이상 작업 확인, 필터 저장, 상세 화면 이동, 담당자 큐 이동, 보고서 다운로드
- 결재/승인: 조회와 필터링은 결재 없이 가능하지만 상태 변경이나 다운로드는 권한과 사유를 확인한다.
- 연계 메뉴: 관리자 계정, 역할/권한, 개인정보 조회 로그, 공통 설정, 승인선 관리, 감사로그
- 권한/감사: 조회 사유, 검색 조건, 마스킹 해제, 다운로드, 등록/수정/삭제, 승인/반려, 실행 결과와 실패 사유를 관리자 감사로그로 남긴다.

## 설계 방식 제안

- 목록, 상세, 처리 이력, 변경 이력을 기준 구조로 둔다.
- 조회 권한과 처리 권한을 분리한다.
- 변경성 액션은 사유 입력과 감사로그를 필수로 둔다.
- 관련 회원, 거래, 입출금, Case, 보고서 화면으로 이동할 수 있게 연결한다.

## 개발 지침

ADM-SEC-010 화면은 이 문서의 화면 목적, 탭 구성, 설계 방식, 권한/감사 기준을 기준으로 개발한다.
설계와 다르게 구현해야 하는 항목은 변경 사유와 대체 정책을 작업 기록에 남긴다.
