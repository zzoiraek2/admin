# ADM-MKT-003 자산별 보유수량

## 기본 정보

- 메뉴 코드: ADM-MKT-003
- 상단 메뉴: 거래·마켓 관리
- 좌측 1메뉴: 거래정보 조회
- 좌측 2메뉴: 자산별 보유수량
- 상태: 정의됨
- 구 어드민 매핑: 거래 관리 > 자산별 고객 보유수량

## 화면 목적

기준시점별 회원/자산 보유수량과 평가금액을 조회한다.

## 업무 설명

거래와 입출금 결과가 반영된 자산별 보유수량을 조회하는 화면이다. 주문/체결 화면과 중복되지 않도록 거래 행위가 아니라 기준시점 보유량, 주문 묶임 수량, 출금 대기 수량, 사용 가능 수량, 평가금액, 상위 보유자 현황을 중심으로 설계한다.

## 결재 프로세스

- 해당사항 없음

## 탭 구성

- 자산 요약
- 회원별 보유
- 자산별 회원
- 스냅샷 이력

## 탭별 상세 설계

### 자산 요약

- 업무 목적: 자산별 총 보유수량과 평가금액을 기준시점 단위로 확인한다.
- 검색 조건: 기준시점, 자산, 마켓지원여부, 네트워크, 평가통화
- 목록 컬럼: 자산, 티커, 총보유수량, 사용가능수량, 주문묶임수량, 출금대기수량, 평가단가, 평가금액, 보유회원수, 기준시점
- 상세보기 정보: 자산별 보유 구성, 평가금액 산출 기준, 전일 대비 증감, 상위 보유자 링크, 지갑/정산 연계 정보
- 주요 기능: 자산 보유 요약 조회, 평가 기준 확인, 상위 보유자 이동, 스냅샷 비교, 다운로드
- 결재/승인: 해당사항 없음
- 연계 메뉴: ADM-WAL-013 지갑/주소 관리, ADM-WAL-015 핫/콜드 비율 관리, ADM-SETTLE-014 종가/경제적가치
- 권한/감사: 자산별 조회, 다운로드, 상세 이동 이력을 감사로그로 남긴다.

### 회원별 보유

- 업무 목적: 특정 회원의 자산별 보유수량을 조회해 민원, 제한, 정산 확인에 활용한다.
- 검색 조건: 회원번호, 기준시점, 자산, 보유수량 범위, 제한/락업 여부
- 목록 컬럼: 회원번호, 자산, 총보유수량, 사용가능수량, 주문묶임수량, 출금대기수량, 제한수량, 평가금액, 기준시점
- 상세보기 정보: 회원 자산 구성, 보유수량 구성 사유, 관련 주문/체결/입출금 링크, 제한/락업 상태, 최근 변동
- 주요 기능: 회원별 보유 조회, 자산별 상세 이동, 회원 기본조회 이동, 관련 주문/입출금 이동
- 결재/승인: 해당사항 없음
- 연계 메뉴: ADM-MEM-001 회원 기본조회, ADM-MKT-001 거래 요청 내역, ADM-MKT-002 체결 내역, ADM-WAL-001 입출금 조회
- 권한/감사: 회원 단위 조회와 마스킹 해제, 다운로드 이력을 감사로그로 남긴다.

### 자산별 회원

- 업무 목적: 특정 자산을 보유한 회원 목록과 집중도를 확인한다.
- 검색 조건: 자산, 기준시점, 보유수량 범위, 평가금액 범위, 회원상태, KYC상태
- 목록 컬럼: 자산, 회원번호, 총보유수량, 사용가능수량, 주문묶임수량, 출금대기수량, 평가금액, 보유비중, 회원상태
- 상세보기 정보: 자산별 보유 회원 분포, 상위 보유자, 보유 집중도, 회원별 상세 링크
- 주요 기능: 자산별 회원 조회, 상위 보유자 정렬, 보유비중 확인, 회원 상세 이동, 다운로드
- 결재/승인: 해당사항 없음
- 연계 메뉴: ADM-MEM-001 회원 기본조회, ADM-MKT-005 거래량 현황, ADM-WAL-015 핫/콜드 비율 관리
- 권한/감사: 자산별 회원 조회, 정렬/필터, 다운로드 이력을 감사로그로 남긴다.

### 스냅샷 이력

- 업무 목적: 일자 또는 특정 시점 기준 보유수량 변화를 추적한다.
- 검색 조건: 기간, 기준시점, 자산, 회원번호, 변동률 범위
- 목록 컬럼: 기준시점, 자산, 회원번호, 이전총보유수량, 현재총보유수량, 증감수량, 증감률, 평가금액, 생성상태
- 상세보기 정보: 스냅샷 생성 기준, 변동 원인 후보, 관련 주문/체결/입출금 링크, 원천 데이터 상태
- 주요 기능: 스냅샷 조회, 기간 비교, 변동 큰 대상 필터링, 원천 화면 이동, 다운로드
- 결재/승인: 해당사항 없음
- 연계 메뉴: ADM-MKT-002 체결 내역, ADM-WAL-001 입출금 조회, ADM-SETTLE-001 정산 대시보드
- 권한/감사: 스냅샷 조회와 다운로드 이력을 감사로그로 남긴다.

## 설계 방식 제안

- 이 화면은 보유수량 조회 화면이며 주문/체결 원장과 지갑 실물 보유량을 연결하는 조회 허브로 설계한다.
- 자산 보유수량의 원인 추적은 주문, 체결, 입출금, 정산 메뉴로 이동해 확인한다.
- 수량 보정이나 제한 설정은 이 화면에서 수행하지 않는다.

## 개발 지침

ADM-MKT-003 화면은 기준시점별 보유수량 조회와 연계 이동을 기준으로 개발한다. 보유수량을 직접 수정하는 기능은 이 메뉴에 포함하지 않는다.
설계와 다르게 구현해야 하는 항목은 변경 사유와 대체 정책을 작업 기록에 남긴다.
