# ADM-SEC-002 MFA/IP 제한

## 기본 정보

- 메뉴 코드: ADM-SEC-002
- 상단 메뉴: 보안·관리자통제
- 좌측 1메뉴: 관리자 계정
- 좌측 2메뉴: MFA/IP 제한
- 상태: 준비중
- 구 어드민 매핑: 신규 정책 정의

## 화면 목적

관리자 로그인 보안 관리

## 업무 설명

MFA 등록, IP 제한, 접속위치, 예외승인

## 탭 구성

- 관리자 요약
- 접근·보안 정책
- 승인·변경
- 접속·작업 이력
- 감사로그


## 탭별 상세 설계

### 관리자 요약

- 업무 목적: MFA/IP 제한의 관리자 요약 탭은 보안 관리자, 시스템 관리자, 감사 담당자, 승인권자가 관리자 로그인 보안 관리 업무를 관리자 요약 관점에서 판단하고 다음 처리 화면으로 자연스럽게 이동하도록 설계한다.
- 검색 조건: 관리자ID, 역할, 권한, IP, 기간, 작업유형, 승인상태, 이상징후, 관리자 요약 상태/구분
- 목록 컬럼: 관리자, 역할/권한, 대상 메뉴, 작업유형, 사유, IP/기기, 승인상태, 작업시각, 관리자 요약 주요 판단값
- 상세보기 정보: 권한 범위, 접근 이력, 작업 전후 값, 조회 사유, 마스킹 해제 여부, 승인/반려 이력, 관리자 요약별 판단 기준, 운영자 메모, 관련 증빙
- 주요 기능: 권한 부여/회수 상신, 세션 종료, 설정 변경, 로그 다운로드, 이상 작업 확인, 알림 확인, 추이 비교, 드릴다운 조회, 담당 업무 큐 이동, 설정 초안 저장, 영향도 검증, 적용 예약, 비활성 처리, 롤백 요청, 위험 근거 확인, 차단/해제 요청, 보류 처리, 후속 Case 연결
- 결재/승인: 등록, 변경, 해제, 제한 적용, 대량 처리, 외부 보고 값 확정은 결재 상신 후 승인 완료 시 실행한다.
- 연계 메뉴: 관리자 계정, 역할/권한, 개인정보 조회 로그, 공통 설정, 승인선 관리, 감사로그
- 권한/감사: 조회 사유, 검색 조건, 마스킹 해제, 다운로드, 등록/수정/삭제, 승인/반려, 실행 결과와 실패 사유를 관리자 감사로그로 남긴다.

## 설계 방식 제안

- 목록, 상세, 처리 이력, 변경 이력을 기준 구조로 둔다.
- 조회 권한과 처리 권한을 분리한다.
- 변경성 액션은 사유 입력과 감사로그를 필수로 둔다.
- 관련 회원, 거래, 입출금, Case, 보고서 화면으로 이동할 수 있게 연결한다.

## 개발 지침

ADM-SEC-002 화면은 이 문서의 화면 목적, 탭 구성, 설계 방식, 권한/감사 기준을 기준으로 개발한다.
설계와 다르게 구현해야 하는 항목은 변경 사유와 대체 정책을 작업 기록에 남긴다.
