# ADM-MEM-008 KYC 심사 대기

## 기본 정보

- 메뉴 코드: ADM-MEM-008
- 상단 메뉴: 회원·고객확인
- 좌측 1메뉴: 고객확인 관리
- 좌측 2메뉴: KYC 심사 대기
- 상태: 정의됨
- 구 어드민 매핑: 회원관리 회원관리 > 고객확인 > 고객확인(KYC) 확인 

## 화면 목적

신규·재이행 KYC 심사 처리

## 업무 설명

신분증 OCR, 안면인증, 고객정보, 직업, 거래목적, 증빙 확인

## 탭 구성

- 신청·증빙
- OCR·안면인증
- 검토·보완
- 승인·반려
- 자동 제한·통지
- 감사로그


## 탭별 상세 설계

### 신청·증빙

- 업무 목적: 심사 대상 목록에서 선택된 회원의 KYC 신청 정보와 제출 증빙을 확인하는 첫 상세 탭이다. 대기열 판단값은 화면 기본 목록에서 보고, 이 탭에서는 선택 건의 심사 가능 여부를 확인한다.
- 검색 조건: 접수일시, 심사유형, 회원번호, KYC 단계, 제출상태, 증빙유형, 만료 여부, 보완 요청 여부
- 목록 컬럼: 회원번호, 심사유형, 제출상태, 증빙유형, OCR 결과, 유효기간, 보완 필요, 최근 제출일
- 상세보기 정보: 회원 요약, 신청 경로, 심사 유형, 제출 완료 여부, 신분증 이미지, 안면인증 캡처, 직업/거래목적, 자금원천 증빙, 제출 변경 이력
- 주요 기능: 증빙 열람, 마스킹 해제 요청, 제출 완전성 확인, 보완 필요 항목 체크, 보완 요청 초안 작성, 관련 회원 화면 이동
- 결재/승인: 민감증빙 다운로드와 마스킹 해제는 별도 권한과 사유 입력을 요구한다. 대기열 배정 자체는 화면 목록의 운영 기능으로 분리한다.
- 연계 메뉴: 회원 기본조회, 고객확인 현황조회, 로그인·보안 조회, 입출금 제한, 상담 이력, 감사로그
- 권한/감사: 심사자, 승인자, 조회 사유, 증빙 열람, 마스킹 해제, 승인/반려, 고객 통지와 상태 반영 결과를 관리자 감사로그로 남긴다.

### OCR·안면인증

- 업무 목적: 자동 검증 결과와 실패 사유를 확인하고 수동 판정이 필요한 케이스를 분리하는 탭이다.
- 검색 조건: 회원번호, OCR 결과, 안면인증 결과, 실패코드, 수동검토 필요 여부
- 목록 컬럼: 회원번호, OCR 일치율, 안면일치율, 실패코드, 자동판정, 수동검토 여부, 재시도 횟수
- 상세보기 정보: OCR 원문/수정값 비교, 얼굴 일치율, 실패 원인, 재시도 이력, 기기/IP 정보
- 주요 기능: 수동검토 지정, 재제출 요청, 자동판정 근거 확인, 실패코드 메모 등록
- 결재/승인: 자동 실패를 수동 통과 처리하는 경우 승인자 확인과 사유 기록이 필수다.
- 연계 메뉴: 회원 기본조회, 고객확인 현황조회, 로그인·보안 조회, 입출금 제한, 상담 이력, 감사로그
- 권한/감사: 심사자, 승인자, 조회 사유, 증빙 열람, 마스킹 해제, 승인/반려, 고객 통지와 상태 반영 결과를 관리자 감사로그로 남긴다.

### 검토·보완

- 업무 목적: 심사자가 제출 정보와 증빙을 검토하고 보완 요청, 보류, 내부 검토 의견을 남기는 탭이다.
- 검색 조건: 회원번호, 검토상태, 보완유형, 담당자, 보완기한, 위험등급
- 목록 컬럼: 회원번호, 검토상태, 보완항목, 보완기한, 담당자, 위험등급, 최근 코멘트
- 상세보기 정보: 검토 체크리스트, 보완 요청 항목, 내부 의견, 고객 안내 문구, 관련 상담 이력
- 주요 기능: 보완 요청, 내부 메모, 보류 처리, 추가 증빙 요청, 검토 완료 처리
- 결재/승인: 보완 요청은 담당자 권한으로 가능하나 고위험 보류나 예외 인정은 승인 흐름을 사용한다.
- 연계 메뉴: 회원 기본조회, 고객확인 현황조회, 로그인·보안 조회, 입출금 제한, 상담 이력, 감사로그
- 권한/감사: 심사자, 승인자, 조회 사유, 증빙 열람, 마스킹 해제, 승인/반려, 고객 통지와 상태 반영 결과를 관리자 감사로그로 남긴다.

### 승인·반려

- 업무 목적: KYC 승인, 반려, 조건부 승인, 재심사 요청을 최종 처리하고 KYC 상태 반영 결과를 확인하는 탭이다.
- 검색 조건: 회원번호, 최종판정, 승인자, 반려사유, 처리일시, 고위험 여부, 고객안내 상태
- 목록 컬럼: 회원번호, 최종판정, 판정사유, 승인자, 처리일시, 고객안내 상태, KYC 상태 반영 결과
- 상세보기 정보: 최종 판정 근거, 체크리스트 결과, 반려 사유, 고객 안내 문구, 승인 전후 KYC 상태, 자동 제한/해제 연계 결과
- 주요 기능: 승인, 반려, 조건부 승인, 재심사 요청, 고객 통지 예약, KYC 상태 반영 결과 확인
- 결재/승인: 고위험, PEP, 제재 의심, 수동 예외 승인 건은 2단계 승인 또는 준법 검토 후 확정한다.
- 연계 메뉴: 회원 기본조회, 고객확인 현황조회, 로그인·보안 조회, 입출금 제한, 상담 이력, 감사로그
- 권한/감사: 심사자, 승인자, 조회 사유, 증빙 열람, 마스킹 해제, 승인/반려, 고객 통지와 상태 반영 결과를 관리자 감사로그로 남긴다.

### 자동 제한·통지

- 업무 목적: KYC 미완료, 재이행 만료, 반려 등에 따라 정책 기반으로 적용되는 자동 제한, 자동 해제, 고객 통지 스케줄을 모니터링하는 탭이다. 운영자가 직접 제한을 적용하거나 해제하는 탭이 아니다.
- 검색 조건: 회원번호, KYC 상태, 자동 제한 상태, 제한 사유, 통지상태, 재이행 기한, 자동 해제 여부
- 목록 컬럼: 회원번호, KYC 상태, 자동 제한 범위, 제한 사유, 통지 스케줄, 최근 통지일, 재이행 기한, 자동 해제 반영상태
- 상세보기 정보: 제한 전후 상태, 정책 트리거, 통지 발송 이력, KYC 완료 후 해제 반영 결과, 실패/예외 Case, 스케줄 재시도 이력
- 주요 기능: 자동 제한 상태 조회, 통지 스케줄 확인, 발송 이력 확인, 해제 반영 결과 확인, 실패/예외 Case 이동
- 결재/승인: 자동 제한과 자동 해제는 승인 액션이 아니라 정책에 따라 반영된다. 예외 처리, 수동 보정, 정책 변경만 승인 대상으로 둔다.
- 연계 메뉴: 회원 기본조회, 고객확인 현황조회, 로그인·보안 조회, 입출금 제한, 상담 이력, 감사로그
- 권한/감사: 심사자, 승인자, 조회 사유, 증빙 열람, 마스킹 해제, 승인/반려, 고객 통지와 상태 반영 결과를 관리자 감사로그로 남긴다.

### 감사로그

- 업무 목적: KYC 심사 과정의 조회, 증빙 열람, 판정, 보완 요청, 승인/반려 이력을 감사 기준으로 추적한다.
- 검색 조건: 회원번호, 관리자ID, 작업유형, 기간, IP, 승인결과
- 목록 컬럼: 작업일시, 관리자, 작업유형, 대상 회원, 사유, IP, 변경 전후 값
- 상세보기 정보: 조회 사유, 마스킹 해제 사유, 판정 전후 상태, 승인자, 고객 통지 여부
- 주요 기능: 감사 로그 조회, 변경 전후 비교, 로그 다운로드, 관련 승인 건 이동
- 결재/승인: 감사 로그는 수정할 수 없으며 다운로드는 별도 권한과 사유 입력을 요구한다.
- 연계 메뉴: 회원 기본조회, 고객확인 현황조회, 로그인·보안 조회, 입출금 제한, 상담 이력, 감사로그
- 권한/감사: 심사자, 승인자, 조회 사유, 증빙 열람, 마스킹 해제, 승인/반려, 고객 통지와 상태 반영 결과를 관리자 감사로그로 남긴다.

## 설계 방식 제안

- 목록, 상세, 처리 이력, 변경 이력을 기준 구조로 둔다.
- 조회 권한과 처리 권한을 분리한다.
- 변경성 액션은 사유 입력과 감사로그를 필수로 둔다.
- 관련 회원, 거래, 입출금, Case, 보고서 화면으로 이동할 수 있게 연결한다.

## KYC 심사 대기 목록 기준

- 이 화면의 기본 목록이 KYC 심사 대기열 역할을 한다. 대기열은 별도 탭으로 분리하지 않는다.
- 기본 목록 컬럼은 접수일시, 회원번호, 심사유형, KYC 단계, 제출상태, 위험등급, 담당자, SLA, 보완횟수, 처리상태를 우선으로 둔다.
- 운영자는 기본 목록에서 심사 대상을 선택하고, 우측 또는 하단 상세 영역의 탭에서 신청 증빙, OCR·안면인증, 검토·보완, 승인·반려, 자동 제한·통지 정보를 확인한다.
- 담당자 배정, 우선순위, SLA 초과 여부는 목록의 필터와 정렬 기준으로 제공하고, 탭 안에서 별도 업무로 중복 구현하지 않는다.
- KYC 미완료, 재이행 만료, 반려에 따른 제한과 KYC 완료 후 해제는 정책 기반 자동 반영을 원칙으로 하며, 예외·실패·수동 보정만 별도 승인 대상으로 관리한다.

## 개발 지침

ADM-MEM-008 화면은 이 문서의 화면 목적, 탭 구성, 설계 방식, 권한/감사 기준을 기준으로 개발한다.
설계와 다르게 구현해야 하는 항목은 변경 사유와 대체 정책을 작업 기록에 남긴다.
