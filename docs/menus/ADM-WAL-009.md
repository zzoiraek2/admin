# ADM-WAL-009 출금 재처리/실패관리

## 기본 정보

- 메뉴 코드: ADM-WAL-009
- 상단 메뉴: 입출금·지갑
- 좌측 1메뉴: 출금 운영
- 좌측 2메뉴: 출금 재처리/실패관리
- 상태: 준비중
- 구 어드민 매핑: 신규 정책 정의

## 화면 목적

출금 실패 건의 원인을 확인하고 자동 재시도 또는 수동 재처리 대상 건을 관리한다.

## 업무 설명

지갑 서버 오류, 수수료 부족, 브로드캐스트 실패, 네트워크 장애, TX 생성 실패 등으로 실패한 출금 건을 관리한다. 자동 재시도는 모니터링하고, 수동 재처리, TX 재생성, 수수료 재계산, 출금 취소/대체 처리는 결재 후 실행한다.

## 결재 프로세스

- 수동 재처리: TX 재생성, 수수료 재계산, 출금 취소, 대체 처리, 수동 재시도는 승인 완료 후 실행한다.

## 탭 구성

- 실패 큐
- 실패 원인
- 자동 재시도
- 수동 재처리
- 처리 이력

## 탭별 상세 설계

### 실패 큐

- 업무 목적: 실패 또는 재처리 필요 출금 건을 확인한다.
- 검색 조건: 기간, 회원번호, 자산, 실패유형, 처리상태, 재시도횟수
- 목록 컬럼: 실패시각, 출금번호, 회원번호, 자산, 금액, 실패유형, 실패사유, 재시도횟수, 처리상태
- 상세보기 정보: 출금 요청값, 지갑 서버 응답, 실패 로그, 잔액/수수료 상태, 다음 처리 후보
- 주요 기능: 실패 건 조회, 원인 상세 이동, 자동/수동 처리 구분
- 결재/승인: 해당사항 없음
- 연계 메뉴: ADM-WAL-007 가상자산 출금 현황, ADM-WAL-017 지갑잔고 대사
- 권한/감사: 실패 큐 조회와 상세 이동 이력을 감사로그로 남긴다.

### 실패 원인

- 업무 목적: 출금 실패 원인을 유형화하고 재처리 가능 여부를 판단한다.
- 검색 조건: 실패유형, 자산, 네트워크, 지갑서버, 재처리가능 여부
- 목록 컬럼: 출금번호, 실패유형, 원인후보, 지갑서버응답, 수수료상태, 재처리가능 여부, 담당자
- 상세보기 정보: 서버 응답 전문, 수수료 산출, 브로드캐스트 로그, 노드 상태, 운영자 메모
- 주요 기능: 원인 분류, 재처리 가능 표시, 메모 등록, 수동 재처리 이동
- 결재/승인: 해당사항 없음
- 연계 메뉴: ADM-WAL-017 지갑잔고 대사, ADM-DASH-001 운영 대시보드
- 권한/감사: 원인 분류와 메모 이력을 감사로그로 남긴다.

### 자동 재시도

- 업무 목적: 자동 재시도 대상과 결과를 모니터링한다.
- 검색 조건: 기간, 자산, 재시도상태, 재시도횟수, 실패유형
- 목록 컬럼: 출금번호, 자산, 금액, 재시도횟수, 재시도상태, 최근시도시각, 결과, 실패사유
- 상세보기 정보: 자동 재시도 조건, 시도 이력, 성공/실패 결과, 다음 시도 예정
- 주요 기능: 자동 재시도 상태 조회, 실패 반복 대상 필터링, 수동 재처리 이동
- 결재/승인: 해당사항 없음
- 연계 메뉴: ADM-WAL-007 가상자산 출금 현황, ADM-WAL-009 출금 재처리/실패관리
- 권한/감사: 자동 재시도 조회와 이동 이력을 감사로그로 남긴다.

### 수동 재처리

- 업무 목적: 자동 처리되지 않는 실패 건을 승인 후 수동 재처리한다.
- 검색 조건: 기간, 승인상태, 처리상태, 실패유형, 회원번호
- 목록 컬럼: 상신번호, 출금번호, 회원번호, 자산, 금액, 처리유형, 승인상태, 실행상태, 처리시각
- 상세보기 정보: 재처리 사유, 변경 전후 값, 수수료 재계산, TX 재생성 요청값, 승인 의견, 실행 결과
- 주요 기능: 결재 상신, 승인, 반려, 수동 재시도, TX 재생성, 출금 취소/대체 처리
- 결재/승인: TX 재생성, 수수료 재계산, 출금 취소, 대체 처리, 수동 재시도는 승인 완료 후 실행한다.
- 연계 메뉴: ADM-WAL-007 가상자산 출금 현황, ADM-SEC-007 감사로그
- 권한/감사: 상신, 승인/반려, 재처리 실행, 실패 사유를 감사로그로 남긴다.

- 업무 목적: 출금 실패와 재처리 결과를 추적한다.
- 검색 조건: 기간, 출금번호, 회원번호, 처리유형, 처리결과

- 상세보기 정보: 실패 원인, 재처리 과정, 승인 이력, 실행 결과, 감사로그
- 주요 기능: 처리 이력 조회, 출금 현황 이동, 감사로그 이동, 다운로드
- 결재/승인: 해당사항 없음
- 연계 메뉴: ADM-WAL-007 가상자산 출금 현황, ADM-SEC-007 감사로그

## 설계 방식 제안

- 자동 재시도와 수동 재처리를 분리한다.
- 수동 재처리는 고객 자산과 외부 전송에 영향을 주므로 결재 후 실행한다.
- 출금 실패 원인은 지갑잔고 대사, 노드 상태, 원장 상태와 연결해 확인한다.

ADM-WAL-009 화면은 출금 실패 원인 확인, 자동 재시도 모니터링, 수동 재처리 결재 흐름을 기준으로 개발한다.
설계와 다르게 구현해야 하는 항목은 변경 사유와 대체 정책을 작업 기록에 남긴다.
