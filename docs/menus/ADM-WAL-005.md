# ADM-WAL-005 입금 미반영/지연 관리

## 기본 정보

- 메뉴 코드: ADM-WAL-005
- 상단 메뉴: 입출금·지갑
- 좌측 1메뉴: 입금 운영
- 좌측 2메뉴: 입금 미반영/지연 관리
- 상태: 준비중
- 구 어드민 매핑: 신규 정책 정의

## 화면 목적

입금 미반영 또는 지연 건의 원인을 확인하고 수동 반영이 필요한 건을 승인 절차로 처리한다.

## 업무 설명

미매칭 주소, 태그 누락, 컨펌 부족, 노드 장애, 원장 반영 실패 등으로 고객 잔고에 반영되지 않은 입금 건을 관리한다. 수동 입금 반영, 회원 매칭 확정, 금액 보정은 고객 잔고에 직접 영향을 주므로 결재 후 처리한다.

## 결재 프로세스

- 수동 반영 검토: 고객 잔고 반영 대상과 금액을 확정하는 단계이므로 결재 상신 전 검토 근거를 남긴다.
- 반영 승인·처리: 수동 입금 반영, 회원 매칭 확정, 금액 보정은 승인 완료 후 실행한다.

## 탭 구성

- 미반영 큐
- 원인 분류
- 수동 반영 검토
- 반영 승인·처리
- 처리 이력

- 검색 조건: 기간, 자산, 네트워크, TXID, 미반영사유, 지연시간, 처리상태
- 목록 컬럼: 탐지시각, 자산, 네트워크, TXID, 금액, 입금주소, 태그/메모, 미반영사유, 지연시간, 처리상태
- 상세보기 정보: 입금 탐지 정보, 컨펌 상태, 주소/태그, 회원 매칭 후보, 원장 반영 상태
- 주요 기능: 대상 조회, 원인 분류 이동, 관련 TX/회원/주소 조회

- 연계 메뉴: ADM-WAL-004 가상자산 입금 현황, ADM-WAL-013 지갑/주소 관리

## 설계 방식 제안

- 조회/원인 분류와 고객 잔고 반영 실행을 분리한다.
- 수동 반영, 회원 매칭 확정, 금액 보정은 반드시 결재 후 실행한다.
- 자동 반영 대상은 이 메뉴에서 임의로 바꾸지 않고 원천 입금 로직을 따른다.

## 개발 지침

ADM-WAL-005 화면은 미반영 입금의 원인 확인, 수동 반영 검토, 승인 처리 흐름을 기준으로 개발한다. 승인 없는 잔고 반영은 허용하지 않는다.
설계와 다르게 구현해야 하는 항목은 변경 사유와 대체 정책을 작업 기록에 남긴다.
