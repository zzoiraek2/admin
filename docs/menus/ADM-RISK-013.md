- 목록 컬럼: 발생일시, 회원/대상, 사유코드, 차단범위, 건수/금액, 상태, 승인단계, 근거 문서, VASP 요약 주요 판단값

- 주요 기능: Case 검토, 차단 등록/해제 상신, 증빙 첨부, 보고 집계 확인, 원천 로그 추적, 알림 확인, 추이 비교, 드릴다운 조회, 담당 업무 큐 이동, 설정 초안 저장, 영향도 검증, 적용 예약, 비활성 처리, 롤백 요청, 위험 근거 확인, 차단/해제 요청, 보류 처리, 후속 Case 연결
- 결재/승인: 등록, 변경, 해제, 제한 적용, 대량 처리, 외부 보고 값 확정은 결재 상신 후 승인 완료 시 실행한다.

- 목록, 상세, 처리 이력, 변경 이력을 기준 구조로 둔다.
- 조회 권한과 처리 권한을 분리한다.
- 변경성 액션은 사유 입력과 감사로그를 필수로 둔다.
- 관련 회원, 거래, 입출금, Case, 보고서 화면으로 이동할 수 있게 연결한다.

## KYT VASP 차단 관리 정책

- 이 메뉴는 미신고거래소, OTC 의심 채널, 차단 대상 VASP를 KYT 기준으로 관리하는 화면이다.
- 입력 항목은 `No`, `VASP명`, `Cluster Name`, `Site`, `Status`, `내부 비고`를 기준으로 둔다.
- 등록, 수정, 비활성, 검증상태 변경, 차단목록 반영은 승인과 감사로그를 남긴다.
- 차단목록 반영 이력은 반영 요청자, 승인자, 반영 시점, 반영 대상, 변경 전후 값을 조회할 수 있어야 한다.
- 예시 데이터는 운영 문서에 직접 고정하지 않고 차단 대상 목록의 입력 형식과 검증 기준만 문서화한다.
