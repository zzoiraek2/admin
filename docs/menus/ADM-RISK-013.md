## 탭별 상세 설계

### VASP 요약

- 업무 목적: KYT 기준에서 차단 또는 주의 대상으로 관리할 VASP/거래소/OTC 채널의 현황을 요약하는 탭이다.
- 검색 조건: VASP명, Status, 검증상태, 반영상태, 변경일, 담당자
- 목록 컬럼: No, VASP명, Cluster Name, Site, Status, 검증상태, 반영상태, 내부 비고
- 상세보기 정보: 등록 현황, 검증 필요 건, 차단목록 반영 여부, 최근 변경, 승인 대기 건
- 주요 기능: 목록 조회, 신규 등록 진입, 검증 필요 필터, 반영 이력 이동
- 결재/승인: 해당사항 없음
- 연계 메뉴: KYT Alert, 입출금 VASP 검증, 차단 등록, 차단 사유코드, 감사로그
- 권한/감사: 등록, 수정, 비활성, 검증상태 변경, 차단목록 반영, 승인/반려 이력을 감사로그로 남긴다.

### 식별정보 관리

- 업무 목적: VASP명, Cluster Name, Site, Status, 내부 비고를 기준으로 식별 정보를 등록·수정하는 탭이다.
- 검색 조건: VASP명, Cluster Name, Site, Status, 중복 여부, 활성상태
- 목록 컬럼: No, VASP명, Cluster Name, Site, Status, 활성상태, 내부 비고, 최종 변경자
- 상세보기 정보: VASP 식별명, alias, 사이트 URL, 텔레그램/외부 채널, Status, 내부 비고, 중복 후보
- 주요 기능: 등록, 수정, 비활성, 중복 검사, alias 병합 요청, 사이트 검증 요청
- 결재/승인: 신규 등록, 비활성, alias 병합은 승인 후 적용한다.
- 연계 메뉴: KYT Alert, 입출금 VASP 검증, 차단 등록, 차단 사유코드, 감사로그
- 권한/감사: 등록, 수정, 비활성, 검증상태 변경, 차단목록 반영, 승인/반려 이력을 감사로그로 남긴다.

### 검증·위험상태

- 업무 목적: 등록된 VASP 식별정보가 차단 또는 주의 대상으로 볼 근거가 충분한지 검증하는 탭이다.
- 검색 조건: 검증상태, 위험등급, 근거유형, 담당자, 최근 검증일
- 목록 컬럼: VASP명, Cluster Name, Site, 검증상태, 위험등급, 근거유형, 담당자
- 상세보기 정보: Chainalysis/KYT 근거, 사이트 접근 결과, 내부 메모, 미신고 의심 근거, 검증 이력
- 주요 기능: 검증 의견 등록, 위험등급 변경, 근거 첨부, 보완 요청, 검증 완료
- 결재/승인: 고위험 또는 차단 대상 판정은 승인 로그와 근거를 남긴다.

### 차단목록 반영

- 업무 목적: 검증 완료된 VASP 정보를 KYT 차단목록 또는 주의목록에 반영하는 탭이다.
- 검색 조건: 반영상태, 반영일, 반영자, 실패여부, VASP명, Status
- 목록 컬럼: VASP명, Cluster Name, 반영목록, 반영상태, 반영일, 실패사유, 승인상태
- 상세보기 정보: 반영 대상, 반영 전후 값, 실패 로그, 롤백 조건, 관련 KYT 정책
- 주요 기능: 반영 상신, 승인 결과 반영, 실패 재시도, 비활성 반영, 반영 이력 확인
- 결재/승인: 차단목록 반영은 승인 완료 후 실행하고 실패 시 재시도와 롤백 이력을 남긴다.
- 연계 메뉴: KYT Alert, 입출금 VASP 검증, 차단 등록, 차단 사유코드, 감사로그
- 권한/감사: 등록, 수정, 비활성, 검증상태 변경, 차단목록 반영, 승인/반려 이력을 감사로그로 남긴다.

### 승인·변경 이력

- 검색 조건: 기간, 관리자ID, 작업유형, 승인상태, VASP명, IP
- 목록 컬럼: 작업일시, 관리자, 작업유형, VASP명, 변경 전후 값, 승인결과, IP

- 주요 기능: 승인, 반려, 보완 요청, 변경 전후 비교, 감사 로그 다운로드
- 결재/승인: 해당사항 없음

## KYT VASP 차단 관리 정책

- 이 메뉴는 미신고거래소, OTC 의심 채널, 차단 대상 VASP를 KYT 기준으로 관리하는 화면이다.
- 입력 항목은 `No`, `VASP명`, `Cluster Name`, `Site`, `Status`, `내부 비고`를 기준으로 둔다.
- 등록, 수정, 비활성, 검증상태 변경, 차단목록 반영은 승인과 감사로그를 남긴다.
- 차단목록 반영 이력은 반영 요청자, 승인자, 반영 시점, 반영 대상, 변경 전후 값을 조회할 수 있어야 한다.
- 예시 데이터는 운영 문서에 직접 고정하지 않고 차단 대상 목록의 입력 형식과 검증 기준만 문서화한다.
