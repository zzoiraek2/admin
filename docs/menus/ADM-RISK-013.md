# ADM-RISK-013 KYT VASP 차단 관리

## 기본 정보

- 메뉴 코드: ADM-RISK-013
- 상단 메뉴: 준법·FDS 관리
- 좌측 1메뉴: KYT/Chainalysis
- 좌측 2메뉴: KYT VASP 차단 관리
- 상태: 준비중
- 구 어드민 매핑: 신규 정책 정의

## 화면 목적

KYT 기준 VASP 차단 목록 관리

## 업무 설명

No, VASP명, Cluster Name, Site, Status, 내부 비고

## 탭 구성

- VASP 요약
- 식별정보 관리
- 검증·위험상태
- 차단목록 반영
- 승인·변경 이력

## 설계 방식 제안

- 목록, 상세, 처리 이력, 변경 이력을 기준 구조로 둔다.
- 조회 권한과 처리 권한을 분리한다.
- 변경성 액션은 사유 입력과 감사로그를 필수로 둔다.
- 관련 회원, 거래, 입출금, Case, 보고서 화면으로 이동할 수 있게 연결한다.

## KYT VASP 차단 관리 정책

- 이 메뉴는 미신고거래소, OTC 의심 채널, 차단 대상 VASP를 KYT 기준으로 관리하는 화면이다.
- 입력 항목은 `No`, `VASP명`, `Cluster Name`, `Site`, `Status`, `내부 비고`를 기준으로 둔다.
- 등록, 수정, 비활성, 검증상태 변경, 차단목록 반영은 승인과 감사로그를 남긴다.
- 차단목록 반영 이력은 반영 요청자, 승인자, 반영 시점, 반영 대상, 변경 전후 값을 조회할 수 있어야 한다.
- 예시 데이터는 운영 문서에 직접 고정하지 않고 차단 대상 목록의 입력 형식과 검증 기준만 문서화한다.

## 개발 지침

ADM-RISK-013 화면은 이 문서의 화면 목적, 탭 구성, 설계 방식, 권한/감사 기준을 기준으로 개발한다.
설계와 다르게 구현해야 하는 항목은 변경 사유와 대체 정책을 작업 기록에 남긴다.
