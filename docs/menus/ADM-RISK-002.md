# ADM-RISK-002 차단 해제

## 기본 정보

- 상단 메뉴: 준법·FDS 관리
- 좌측 1메뉴: 입출금 차단 관리
- 좌측 2메뉴: 차단 해제
- 상태: 준비중
- 구 어드민 매핑: 신규 정책 정의

## 화면 목적

차단 해제 요청 처리

## 업무 설명

해제사유, 요청자, 승인자, 해제일시, 관련 증빙

## 탭 구성

- 해제 요청
- 원 차단 근거
- 검토·승인
- 재개·통지
- 감사로그

## 설계 방식 제안

- 목록, 상세, 처리 이력, 변경 이력을 기준 구조로 둔다.
- 조회 권한과 처리 권한을 분리한다.
- 변경성 액션은 사유 입력과 감사로그를 필수로 둔다.
- 관련 회원, 거래, 입출금, Case, 보고서 화면으로 이동할 수 있게 연결한다.

## 차단 해제 정책

- 차단 해제는 FDS 차단 고객, 수동 차단 등록 대상자, 지갑주소, 예치금 차단 대상의 해제를 승인 프로세스로 처리하는 화면이다.
- 해제 요청 시 원 차단 사유, 차단 등록 이력, 원천 로그, 증빙, 해제 사유를 함께 보여줘야 한다.
- 해제 승인 전에는 실제 차단 상태가 변경되지 않아야 한다.
- 해제 완료 후에는 재개 일시, 처리자, 승인자, 변경 전후 상태, 통지 여부를 남긴다.
- 반복 차단 대상은 해제 후에도 이전 차단 이력을 조회할 수 있어야 한다.

## 개발 지침

ADM-RISK-002 화면은 이 문서의 화면 목적, 탭 구성, 설계 방식, 권한/감사 기준을 기준으로 개발한다.
설계와 다르게 구현해야 하는 항목은 변경 사유와 대체 정책을 작업 기록에 남긴다.
