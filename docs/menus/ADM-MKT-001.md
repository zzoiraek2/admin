# ADM-MKT-001 거래 요청 내역

## 기본 정보

- 메뉴 코드: ADM-MKT-001
- 상단 메뉴: 거래·마켓 관리
- 좌측 1메뉴: 거래정보 조회
- 좌측 2메뉴: 거래 요청 내역
- 상태: 정의됨
- 구 어드민 매핑: 거래관리 > 거래요청 내역

## 화면 목적

주문장 원장 기준으로 회원 주문의 접수, 대기, 부분체결, 전량체결, 취소, 거절 상태를 조회한다.

## 업무 설명

회원이 제출한 매수/매도 주문을 주문장 기준으로 조회하는 화면이다. 운영자는 주문ID, 회원, 마켓, 주문유형, 가격, 수량, 체결수량, 잔량, 주문상태, 요청/갱신/취소 시각, 거절 사유를 확인하고 체결 내역 또는 회원 화면으로 이동할 수 있어야 한다.

## 결재 프로세스

- 해당사항 없음

## 탭 구성

- 주문장
- 전체 주문
- 취소·거절 주문

## 탭별 상세 설계

### 주문장

- 업무 목적: 현재 유효한 대기 주문과 부분체결 주문을 확인해 특정 마켓의 주문장 상태와 회원 주문 잔량을 빠르게 파악한다.
- 검색 조건: 마켓/페어, 매수/매도, 주문유형, 가격 범위, 회원번호, 주문ID, 주문상태, 요청시각
- 목록 컬럼: 주문ID, 회원번호, 마켓, 매수/매도, 주문유형, 주문가격, 주문수량, 체결수량, 잔량, 주문상태, 요청시각, 갱신시각
- 상세보기 정보: 주문 원장 정보, 체결 진행률, 잔량, 주문 조건, 취소 가능 여부, 연결 체결ID, 회원 기본정보 링크
- 주요 기능: 주문 검색, 주문 상세 조회, 체결 내역 이동, 회원 기본조회 이동, 필터 저장, 권한 기반 다운로드
- 결재/승인: 해당사항 없음
- 연계 메뉴: ADM-MKT-002 체결 내역, ADM-MEM-001 회원 기본조회, ADM-MKT-004 거래 현황
- 권한/감사: 주문 조회, 마스킹 해제, 다운로드, 상세 이동 이력을 관리자 감사로그로 남긴다.

### 전체 주문

- 업무 목적: 기간 기준으로 모든 주문 원장을 조회해 민원, 정산 확인, 거래 흐름 추적에 필요한 주문 이력을 확인한다.
- 검색 조건: 기간, 회원번호, 주문ID, 마켓/페어, 매수/매도, 주문유형, 주문상태, 요청채널
- 목록 컬럼: 주문ID, 회원번호, 마켓, 매수/매도, 주문유형, 주문가격, 주문수량, 체결수량, 취소수량, 주문상태, 요청채널, 요청시각, 최종변경시각
- 상세보기 정보: 최초 주문 정보, 상태 변경 흐름, 체결 연결 정보, 취소/거절 정보, 요청채널, IP/기기 식별값
- 주요 기능: 주문 원장 조회, 상태별 필터링, 주문-체결 연결 확인, 회원별 주문 추적, 다운로드
- 결재/승인: 해당사항 없음
- 연계 메뉴: ADM-MKT-002 체결 내역, ADM-MEM-001 회원 기본조회, ADM-SEC-007 감사로그
- 권한/감사: 기간 검색, 회원 단위 조회, 다운로드, 상세 조회 사유를 감사로그로 남긴다.

### 취소·거절 주문

- 업무 목적: 취소되었거나 시스템/정책 조건으로 거절된 주문을 모아 장애, 정책 제한, 회원 문의 대응 근거를 확인한다.
- 검색 조건: 기간, 마켓/페어, 회원번호, 주문ID, 취소/거절 구분, 거절 사유, 요청채널
- 목록 컬럼: 주문ID, 회원번호, 마켓, 매수/매도, 주문유형, 주문가격, 주문수량, 취소수량, 거절사유, 요청시각, 취소/거절시각
- 상세보기 정보: 취소 요청 주체, 취소 처리 결과, 거절 코드, 거절 메시지, 잔고/호가/주문제한 검증 결과, 관련 로그
- 주요 기능: 취소/거절 주문 조회, 사유 코드 필터링, 관련 회원/주문 이동, 운영 메모 등록, 다운로드
- 결재/승인: 해당사항 없음
- 연계 메뉴: ADM-MEM-001 회원 기본조회, ADM-MKT-004 거래 현황, ADM-RISK-011 KYT Alert
- 권한/감사: 거절 사유 조회, 운영 메모, 다운로드 이력을 감사로그로 남긴다.

## 설계 방식 제안

- 이 화면은 주문장/주문 원장 조회 화면으로 설계하고, 시장감시 판단이나 거래 제한 조치는 포함하지 않는다.
- 체결 결과는 ADM-MKT-002에서 확인하되 주문 상세에서 관련 체결로 이동할 수 있게 한다.
- 다운로드는 별도 탭으로 분리하지 않고 목록 기능으로 제공하며 권한, 사유, 감사로그를 필수로 둔다.

## 개발 지침

ADM-MKT-001 화면은 주문장 원장 조회를 기준으로 개발한다. 주문 상태를 변경하거나 강제 취소하는 기능은 이 메뉴에 포함하지 않는다.
설계와 다르게 구현해야 하는 항목은 변경 사유와 대체 정책을 작업 기록에 남긴다.
