# ADM-MKT-001 거래 요청 내역

## 기본 정보

- 메뉴 코드: ADM-MKT-001
- 상단 메뉴: 거래·마켓 관리
- 좌측 1메뉴: 거래정보 조회
- 좌측 2메뉴: 거래 요청 내역
- 상태: 정의됨
- 구 어드민 매핑: 거래관리 > 거래요청 내역

## 화면 목적

주문 요청 단위 조회

## 업무 설명

주문ID, 회원, 마켓, 매수/매도, 가격, 수량, 요청시간, 실패사유

## 탭 구성

- 거래 요약
- 주문·체결 상세
- 이상거래 징후
- 처리·정정 이력
- 감사로그

## 설계 방식 제안

- 목록, 상세, 처리 이력, 변경 이력을 기준 구조로 둔다.
- 조회 권한과 처리 권한을 분리한다.
- 변경성 액션은 사유 입력과 감사로그를 필수로 둔다.
- 관련 회원, 거래, 입출금, Case, 보고서 화면으로 이동할 수 있게 연결한다.

## 개발 지침

ADM-MKT-001 화면은 이 문서의 화면 목적, 탭 구성, 설계 방식, 권한/감사 기준을 기준으로 개발한다.
설계와 다르게 구현해야 하는 항목은 변경 사유와 대체 정책을 작업 기록에 남긴다.
