# ADM-MKT-001 거래 요청 내역

## 기본 정보

- 메뉴 코드: ADM-MKT-001
- 상단 메뉴: 거래·마켓 관리
- 좌측 1메뉴: 거래정보 조회
- 좌측 2메뉴: 거래 요청 내역
- 상태: 정의됨
- 구 어드민 매핑: 거래관리 > 거래요청 내역

## 화면 목적

주문 요청 단위 조회

## 업무 설명

주문ID, 회원, 마켓, 매수/매도, 가격, 수량, 요청시간, 실패사유

## 탭 구성

- 거래 요약

- 처리·정정 이력
- 감사로그


## 탭별 상세 설계

### 거래 요약

- 업무 목적: 거래 요청 내역의 거래 요약 탭은 거래 운영자, 시장감시 담당자, 상품 담당자, 준법 담당자가 주문 요청 단위 조회 업무를 거래 요약 관점에서 판단하고 다음 처리 화면으로 자연스럽게 이동하도록 설계한다.
- 검색 조건: 기간, 마켓/페어, 회원번호, 주문번호, 체결번호, 주문상태, 이상거래 유형, 거래 요약 상태/구분
- 목록 컬럼: 마켓, 주문/체결 건수, 거래금액, 회원 영향도, 이상징후, 상태, 최근 발생시각, 조치상태, 거래 요약 주요 판단값
- 상세보기 정보: 주문/체결 상세, 호가/가격 영향, 관련 회원, 시장경보, 이상거래 근거, 조치 이력, 거래 요약별 판단 기준, 운영자 메모, 관련 증빙
- 주요 기능: 상세 조회, 시장감시 Case 생성, 거래제한 요청, 공지/알림 연계, 보고서 추출, 알림 확인, 추이 비교, 드릴다운 조회, 담당 업무 큐 이동
- 결재/승인: 조회와 단순 필터링은 결재 없이 가능하되, 상태 변경이나 데이터 추출이 발생하면 승인 흐름으로 전환한다.
- 연계 메뉴: 거래 현황, 회원 기본조회, 시장감시 모니터링, FDS Case, 거래지원 상태, 보고서 센터
- 권한/감사: 조회 사유, 검색 조건, 마스킹 해제, 다운로드, 등록/수정/삭제, 승인/반려, 실행 결과와 실패 사유를 관리자 감사로그로 남긴다.

## 설계 방식 제안

- 목록, 상세, 처리 이력, 변경 이력을 기준 구조로 둔다.
- 조회 권한과 처리 권한을 분리한다.
- 변경성 액션은 사유 입력과 감사로그를 필수로 둔다.
- 관련 회원, 거래, 입출금, Case, 보고서 화면으로 이동할 수 있게 연결한다.

## 개발 지침

설계와 다르게 구현해야 하는 항목은 변경 사유와 대체 정책을 작업 기록에 남긴다.
