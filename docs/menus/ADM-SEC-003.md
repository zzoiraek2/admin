# ADM-SEC-003 역할 관리

## 기본 정보

- 메뉴 코드: ADM-SEC-003
- 상단 메뉴: 보안·관리자통제
- 좌측 1메뉴: 접근권한
- 좌측 2메뉴: 역할 관리
- 상태: 정의됨
- 구 어드민 매핑: 신규 정책 정의

## 화면 목적

RBAC 역할 정의

## 업무 설명

부서별 역할, 직무별 역할, 승인자 역할

## 탭 구성

- 역할목록
- 사용자매핑
- 권한범위
- 변경이력

## 설계 방식 제안

- 목록, 상세, 처리 이력, 변경 이력을 기준 구조로 둔다.
- 조회 권한과 처리 권한을 분리한다.
- 변경성 액션은 사유 입력과 감사로그를 필수로 둔다.
- 관련 회원, 거래, 입출금, Case, 보고서 화면으로 이동할 수 있게 연결한다.

## 개발 지침

ADM-SEC-003 화면은 이 문서의 화면 목적, 탭 구성, 설계 방식, 권한/감사 기준을 기준으로 개발한다.
설계와 다르게 구현해야 하는 항목은 변경 사유와 대체 정책을 작업 기록에 남긴다.
