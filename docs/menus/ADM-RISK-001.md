# ADM-RISK-001 차단 등록

## 기본 정보

- 메뉴 코드: ADM-RISK-001
- 상단 메뉴: 준법·FDS 관리
- 좌측 1메뉴: 입출금 차단 관리
- 좌측 2메뉴: 차단 등록
- 상태: 준비중
- 구 어드민 매핑: 신규 정책 정의

## 화면 목적

준법 목적의 입출금 차단 등록

## 업무 설명

대상, 차단유형, 사유코드, 증빙, 적용범위, 승인


## 결재 프로세스

- 등록 요청: 차단 등록은 승인 완료 후 적용한다. 긴급 등록은 사후 승인 정책과 적용 사유를 별도 기록한다.
- 대상·범위: 대상과 범위 변경은 차단 효력에 직접 영향을 주므로 사유와 승인 이력을 남긴다.
- 사유·증빙: 사유코드 예외 적용이나 증빙 미비 상태의 등록은 승인자 확인 후 진행한다.
- 승인·반영: 승인 완료 전에는 실제 차단을 적용하지 않는다. 반영 실패 시 실패 사유와 재시도 이력을 보관한다.

## 설계 방식 제안

- 목록, 상세, 처리 이력, 변경 이력을 기준 구조로 둔다.
- 조회 권한과 처리 권한을 분리한다.
- 변경성 액션은 사유 입력과 감사로그를 필수로 둔다.
- 관련 회원, 거래, 입출금, Case, 보고서 화면으로 이동할 수 있게 연결한다.

## 차단 등록 정책

- 차단 등록은 고객 대상자, 지갑주소, 가상자산 종목, 예치금 범위 중 어떤 대상을 차단하는지 명확히 지정해야 한다.
- FDS, KYC, KYT, 트래블룰 등 자동 차단은 원천 시스템 이벤트와 사유코드를 함께 저장한다.
- 블랙리스트 지갑주소, 법정 요청, 수사 협조, 동결 요청처럼 외부 근거가 있는 등록은 증빙 문서와 요청 기관 정보를 필수로 둔다.
- 정기점검, 장애, 유의종목 지정처럼 사건 기록이 중심인 항목은 실제 대상 차단 등록과 구분해서 관리한다.
- 수동 등록은 임의 등록을 막기 위해 등록 요청, 검토, 승인, 반영, 감사로그 단계를 둔다.

## 개발 지침

ADM-RISK-001 화면은 이 문서의 화면 목적, 탭 구성, 설계 방식, 권한/감사 기준을 기준으로 개발한다.
설계와 다르게 구현해야 하는 항목은 변경 사유와 대체 정책을 작업 기록에 남긴다.
