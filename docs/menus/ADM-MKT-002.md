# ADM-MKT-002 체결 내역

## 기본 정보

- 메뉴 코드: ADM-MKT-002
- 상단 메뉴: 거래·마켓 관리
- 좌측 1메뉴: 거래정보 조회
- 좌측 2메뉴: 체결 내역
- 상태: 정의됨
- 구 어드민 매핑: 거래관리 > 체결 내역

## 화면 목적

주문이 실제 매칭되어 발생한 체결 원장을 조회한다.

## 업무 설명

체결ID, 주문ID, 회원, 마켓, 체결가, 체결수량, 체결금액, 수수료, maker/taker, 상대 주문, 체결시각, 정산 반영 상태를 확인하는 화면이다. 주문장 조회와 중복되지 않도록 주문의 상태가 아니라 실제 체결 결과와 수수료/정산 연결을 중심으로 설계한다.

## 결재 프로세스

- 해당사항 없음

## 탭 구성

- 체결 목록
- 주문별 체결
- 회원별 체결
- 수수료·정산 연계

## 탭별 상세 설계

### 체결 목록

- 업무 목적: 기간 기준 체결 원장을 조회해 실제 거래 발생 내역을 확인한다.
- 검색 조건: 기간, 마켓/페어, 체결ID, 주문ID, 회원번호, 매수/매도, maker/taker, 정산반영상태
- 목록 컬럼: 체결ID, 주문ID, 회원번호, 마켓, 매수/매도, 체결가, 체결수량, 체결금액, 수수료, maker/taker, 체결시각, 정산반영상태
- 상세보기 정보: 체결 원장, 매칭 주문 정보, 수수료 산출 기준, 정산 반영 결과, 주문장 링크
- 주요 기능: 체결 조회, 체결 상세 조회, 주문 이동, 회원 이동, 정산 반영 상태 확인, 다운로드
- 결재/승인: 해당사항 없음
- 연계 메뉴: ADM-MKT-001 거래 요청 내역, ADM-MEM-001 회원 기본조회, ADM-SETTLE-001 정산 대시보드
- 권한/감사: 체결 조회, 회원 단위 검색, 다운로드 이력을 감사로그로 남긴다.

### 주문별 체결

- 업무 목적: 하나의 주문이 여러 건으로 나뉘어 체결된 경우 체결 흐름을 추적한다.
- 검색 조건: 주문ID, 기간, 회원번호, 마켓/페어, 주문상태
- 목록 컬럼: 주문ID, 주문수량, 누적체결수량, 잔량, 평균체결가, 체결건수, 최초체결시각, 최종체결시각
- 상세보기 정보: 주문 원장 요약, 체결 분할 내역, 체결별 상대 주문, 수수료 합계, 평균체결가 계산 근거
- 주요 기능: 주문별 체결 묶음 조회, 체결 상세 펼침, 주문장 이동, 평균체결가 확인
- 결재/승인: 해당사항 없음
- 연계 메뉴: ADM-MKT-001 거래 요청 내역, ADM-MKT-004 거래 현황
- 권한/감사: 주문ID 기준 조회와 상세 이동 이력을 감사로그로 남긴다.

### 회원별 체결

- 업무 목적: 회원 문의, 이상거래 검토, 정산 확인을 위해 특정 회원의 체결 이력을 조회한다.
- 검색 조건: 기간, 회원번호, 마켓/페어, 매수/매도, 체결금액 범위, 체결ID
- 목록 컬럼: 회원번호, 체결ID, 주문ID, 마켓, 매수/매도, 체결가, 체결수량, 체결금액, 수수료, 체결시각
- 상세보기 정보: 회원 체결 요약, 기간 합계, 주문 연결, 정산 연결, 관련 입출금/자산 변동 링크
- 주요 기능: 회원별 체결 조회, 기간 합계 확인, 회원 기본조회 이동, 관련 주문/정산 이동
- 결재/승인: 해당사항 없음
- 연계 메뉴: ADM-MEM-001 회원 기본조회, ADM-MKT-003 자산별 보유수량, ADM-SETTLE-001 정산 대시보드
- 권한/감사: 회원 단위 조회와 다운로드 사유를 감사로그로 남긴다.

### 수수료·정산 연계

- 업무 목적: 체결 수수료와 정산 반영 상태를 확인해 거래 원장과 정산 원장의 연결 상태를 점검한다.
- 검색 조건: 기간, 마켓/페어, 체결ID, 주문ID, 회원번호, 수수료 통화, 정산반영상태
- 목록 컬럼: 체결ID, 회원번호, 마켓, 체결금액, 수수료율, 수수료금액, 수수료통화, 정산반영상태, 정산반영시각
- 상세보기 정보: 수수료 산출 근거, 멤버십/프로모션 적용 여부, 정산 원장 반영 결과, 오류 사유
- 주요 기능: 수수료 검증, 정산 반영 상태 조회, 정산 화면 이동, 오류 건 필터링
- 결재/승인: 해당사항 없음
- 연계 메뉴: ADM-SETTLE-001 정산 대시보드, ADM-SETTLE-002 정산 원장, ADM-MKT-005 거래량 현황
- 권한/감사: 수수료/정산 상세 조회와 다운로드 이력을 감사로그로 남긴다.

## 설계 방식 제안

- 이 화면은 체결 원장 조회 화면으로 설계하고, 주문 접수/대기 상태는 ADM-MKT-001에서 확인한다.
- 수수료와 정산은 체결 결과의 후속 검증 관점으로만 제공하고 정산 변경 기능은 정산 메뉴에서 처리한다.
- 다운로드는 별도 탭으로 분리하지 않고 목록 기능으로 제공한다.

## 개발 지침

ADM-MKT-002 화면은 체결 원장, 수수료, 정산 반영 상태 조회를 기준으로 개발한다. 원장 값을 수정하는 기능은 이 메뉴에 포함하지 않는다.
설계와 다르게 구현해야 하는 항목은 변경 사유와 대체 정책을 작업 기록에 남긴다.
