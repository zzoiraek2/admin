# ADM-MEM-001 회원 기본조회

## 기본 정보

- 메뉴 코드: ADM-MEM-001
- 상단 메뉴: 회원·고객확인
- 좌측 1메뉴: 회원 통합조회
- 좌측 2메뉴: 회원 기본조회
- 상태: 준비중
- 구 어드민 매핑: 회원관리 회원관리 > 회원관리 > 내국인 회원 검색 

## 화면 목적

회원을 빠르게 검색하고 기본 상태 확인

## 업무 설명

회원번호, 이름, 생년월일, 이메일, 휴대폰, 가입일, 회원상태, KYC상태, 제재상태

## 탭 구성

- 회원 요약
- 거래·입출금 요약
- KYC 상태·이력
- 보안·접속
- 제한·리스크
- 상담·처리이력
- 감사로그


## 탭별 상세 설계

### 회원 요약

- 업무 목적: 회원 기본조회의 회원 요약 탭은 CS, KYC 담당자, 준법/FDS 담당자, 승인권자가 회원을 빠르게 검색하고 기본 상태 확인 업무를 회원 요약 관점에서 판단하고 다음 처리 화면으로 자연스럽게 이동하도록 설계한다.
- 검색 조건: 회원번호, UID, 이름, 이메일, 휴대폰, CI/DI, KYC 상태, 계정상태, 위험등급, 회원 요약 상태/구분
- 목록 컬럼: 회원번호, 이름/식별값, KYC 상태, 계정상태, 거래/입출금 제한, 위험등급, 최근 활동, 담당자, 회원 요약 주요 판단값
- 상세보기 정보: 회원 식별정보, 연락처, KYC 이력, 계정 보안상태, 제한/차단 상태, 상담/처리 이력, 감사로그, 회원 요약별 판단 기준, 운영자 메모, 관련 증빙
- 주요 기능: 회원 360 조회, 관련 입출금/거래 이동, 상태 변경 요청, 제한/해제 상신, 상담 메모 등록, 알림 확인, 추이 비교, 드릴다운 조회, 담당 업무 큐 이동
- 결재/승인: 조회와 단순 필터링은 결재 없이 가능하되, 상태 변경이나 데이터 추출이 발생하면 승인 흐름으로 전환한다.
- 연계 메뉴: 회원 기본조회, KYC 심사 대기, 고객확인 현황조회, 로그인·보안 조회, 입출금 조회, 차단 등록/해제, 상담 이력
- 권한/감사: 조회 사유, 검색 조건, 마스킹 해제, 다운로드, 등록/수정/삭제, 승인/반려, 실행 결과와 실패 사유를 관리자 감사로그로 남긴다.

### KYC 상태·이력

- 업무 목적: 회원 기본조회 안에서 해당 회원의 KYC 현재 상태, 최종 심사 결과, 보완/반려 이력, 재이행 필요 여부를 빠르게 확인하는 조회 탭이다. 실제 승인·반려 처리는 KYC 심사 대기 메뉴로 이동해 수행한다.
- 검색 조건: 회원번호, UID, 이름, CI/DI, KYC 상태, 심사유형, 최종 심사일, 재이행 만료일
- 목록 컬럼: 회원번호, KYC 등급, 현재 상태, 최종 결과, 최종 심사일, 보완/반려 사유, 재이행 필요 여부, 심사 상세 링크
- 상세보기 정보: 최근 KYC 신청 요약, 제출 서류 상태, OCR/안면인증 결과 요약, 직업/거래목적 요약, 반려/보완 이력, 재이행 만료일, 관련 심사 Case 링크
- 주요 기능: KYC 심사 상세 열기, 고객확인 현황조회 이동, 제출서류 요약 열람, 반려/보완 사유 확인, 상담 메모 참고
- 결재/승인: 이 탭에서는 승인, 반려, 보완요청, 상태 변경을 직접 수행하지 않는다. 처리 액션은 ADM-MEM-008 KYC 심사 대기에서 권한과 승인 흐름에 따라 수행한다.
- 연계 메뉴: ADM-MEM-008 KYC 심사 대기, ADM-MEM-009 고객확인 현황조회, ADM-MEM-001 회원 기본조회, 상담 이력, 감사로그
- 권한/감사: KYC 정보 조회, 증빙 열람, 마스킹 해제, 관련 Case 이동 이력을 남기며 처리성 로그는 KYC 심사 대기에서 남긴다.

- 연계 메뉴: 회원 기본조회, KYC 심사 대기, 고객확인 현황조회, 로그인·보안 조회, 입출금 조회, 차단 등록/해제, 상담 이력
- 권한/감사: 조회 사유, 검색 조건, 마스킹 해제, 다운로드, 등록/수정/삭제, 승인/반려, 실행 결과와 실패 사유를 관리자 감사로그로 남긴다.

## 설계 방식 제안

- 목록, 상세, 처리 이력, 변경 이력을 기준 구조로 둔다.
- 조회 권한과 처리 권한을 분리한다.
- 변경성 액션은 사유 입력과 감사로그를 필수로 둔다.
- 관련 회원, 거래, 입출금, Case, 보고서 화면으로 이동할 수 있게 연결한다.

## 회원 기본조회 화면 설계 원칙

- 이 화면의 목적은 회원을 빠르게 식별하고 운영상 즉시 확인해야 하는 상태를 한 번에 판단하는 것이다.
- `연락처`, `가입정보`, `상태`, `태그`는 별도 탭이 아니라 `회원 요약` 탭 안의 필드 그룹이나 요약 카드로 배치한다.
- 검색은 회원번호, UID, 이름, 이메일, 휴대폰, CI/DI, 지갑주소, 계좌, 주문번호, 입출금 TXID 역추적을 고려한다.
- 회원 요약에는 회원번호, KYC 상태, 계정상태, 거래가능 여부, 입출금 제한 여부, 위험등급, 최근 로그인, 가입일을 우선 노출한다.
- 핵심 플래그는 휴면, 탈퇴예정, 잠금, FDS 주의, 제재, KYC 재이행 필요, 고액·이상거래 여부처럼 운영 판단에 필요한 상태를 표시한다.
- 관련 화면 바로가기는 KYC 상세, 입출금, 주문·체결, 지갑, 상담, 제재·차단, 감사로그로 연결한다.

## 개발 지침

ADM-MEM-001 화면은 이 문서의 화면 목적, 탭 구성, 설계 방식, 권한/감사 기준을 기준으로 개발한다.
설계와 다르게 구현해야 하는 항목은 변경 사유와 대체 정책을 작업 기록에 남긴다.
