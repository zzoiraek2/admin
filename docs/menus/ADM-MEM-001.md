# ADM-MEM-001 회원 기본조회

## 기본 정보

- 메뉴 코드: ADM-MEM-001
- 상단 메뉴: 회원·고객확인
- 좌측 1메뉴: 회원 통합조회
- 좌측 2메뉴: 회원 기본조회
- 상태: 준비중
- 구 어드민 매핑: 회원관리 회원관리 > 회원관리 > 내국인 회원 검색 

## 화면 목적

## 결재 프로세스

- 회원 요약: 조회와 필터링은 결재 없이 가능하지만 상태 변경이나 다운로드는 권한과 사유를 확인한다.
- 거래·입출금 요약: 조회와 필터링은 결재 없이 가능하지만 상태 변경이나 다운로드는 권한과 사유를 확인한다.
- KYC 상태·이력: 이 탭에서는 승인, 반려, 보완요청, 상태 변경을 직접 수행하지 않는다. 처리 액션은 ADM-MEM-008 KYC 심사 대기에서 권한과 승인 흐름에 따라 수행한다.
- 보안·접속: 조회는 결재 없이 가능하지만 잠금, 해제, 인증수단 초기화는 승인 또는 별도 권한이 필요하다.
- 제한·리스크: 고위험 또는 예외 판단은 승인 흐름으로 넘기고 검토 근거를 감사로그에 남긴다.

- 업무 목적: 회원 기본조회의 감사로그 탭은 CS, KYC 담당자, 준법/FDS 담당자, 승인권자가 회원 상태와 고객확인 이력 관련 정보를 확인하거나 처리할 때 조회, 변경, 승인, 실행 결과를 사후 감사 기준으로 추적하는 탭이다.
- 검색 조건: 회원번호, UID, 이름, 이메일, 휴대폰, CI/DI, KYC 상태, 계정상태, 위험등급, 기간, 관리자ID, 작업유형, 대상 식별값, 승인결과, IP
- 목록 컬럼: 회원번호, 이름/식별값, KYC 상태, 계정상태, 거래/입출금 제한, 위험등급, 최근 활동, 담당자, 작업일시, 관리자, 작업유형, 대상, 사유, 변경 전후 값, IP, 결과
- 상세보기 정보: 회원 식별정보, 연락처, KYC 이력, 계정 보안상태, 제한/차단 상태, 상담/처리 이력, 감사로그, 조회 사유, 마스킹 해제 사유, 변경 전후 값, 승인 의견, 실행 결과, 실패 사유
- 주요 기능: 회원 360 조회, 관련 입출금/거래 이동, 상태 변경 요청, 제한/해제 상신, 상담 메모 등록, 감사 로그 조회, 변경 전후 비교, 로그 다운로드, 관련 승인 건 이동
- 결재/승인: 감사로그는 수정할 수 없으며 다운로드에는 별도 권한과 사유 입력을 요구한다.
- 연계 메뉴: 회원 기본조회, KYC 심사 대기, 고객확인 현황조회, 로그인·보안 조회, 입출금 조회, 차단 등록/해제, 상담 이력
- 권한/감사: 조회 사유, 검색 조건, 마스킹 해제, 다운로드, 등록/수정/삭제, 승인/반려, 실행 결과와 실패 사유를 관리자 감사로그로 남긴다.

## 설계 방식 제안

- 목록, 상세, 처리 이력, 변경 이력을 기준 구조로 둔다.
- 조회 권한과 처리 권한을 분리한다.
- 변경성 액션은 사유 입력과 감사로그를 필수로 둔다.
- 관련 회원, 거래, 입출금, Case, 보고서 화면으로 이동할 수 있게 연결한다.

## 회원 기본조회 화면 설계 원칙

- 이 화면의 목적은 회원을 빠르게 식별하고 운영상 즉시 확인해야 하는 상태를 한 번에 판단하는 것이다.
- `연락처`, `가입정보`, `상태`, `태그`는 별도 탭이 아니라 `회원 요약` 탭 안의 필드 그룹이나 요약 카드로 배치한다.
- 검색은 회원번호, UID, 이름, 이메일, 휴대폰, CI/DI, 지갑주소, 계좌, 주문번호, 입출금 TXID 역추적을 고려한다.
- 회원 요약에는 회원번호, KYC 상태, 계정상태, 거래가능 여부, 입출금 제한 여부, 위험등급, 최근 로그인, 가입일을 우선 노출한다.
- 핵심 플래그는 휴면, 탈퇴예정, 잠금, FDS 주의, 제재, KYC 재이행 필요, 고액·이상거래 여부처럼 운영 판단에 필요한 상태를 표시한다.
- 관련 화면 바로가기는 KYC 상세, 입출금, 주문·체결, 지갑, 상담, 제재·차단, 감사로그로 연결한다.

## 개발 지침

ADM-MEM-001 화면은 이 문서의 화면 목적, 탭 구성, 설계 방식, 권한/감사 기준을 기준으로 개발한다.
설계와 다르게 구현해야 하는 항목은 변경 사유와 대체 정책을 작업 기록에 남긴다.
