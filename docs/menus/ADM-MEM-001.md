# ADM-MEM-001 회원 기본조회

- 메뉴 코드: ADM-MEM-001
- 상단 메뉴: 회원·고객확인
- 좌측 1메뉴: 회원 통합조회
- 좌측 2메뉴: 회원 기본조회
- 상태: 준비중
- 구 어드민 매핑: 회원관리 회원관리 > 회원관리 > 내국인 회원 검색 

## 화면 목적

회원을 빠르게 검색하고 기본 상태 확인

## 업무 설명

회원번호, 이름, 생년월일, 이메일, 휴대폰, 가입일, 회원상태, KYC상태, 제재상태

## 탭 구성

- 회원 요약
- 거래·입출금 요약
- KYC·심사
- 보안·접속
- 제한·리스크
- 상담·처리이력
- 감사로그

## 설계 방식 제안

- 목록, 상세, 처리 이력, 변경 이력을 기준 구조로 둔다.
- 조회 권한과 처리 권한을 분리한다.
- 변경성 액션은 사유 입력과 감사로그를 필수로 둔다.
- 관련 회원, 거래, 입출금, Case, 보고서 화면으로 이동할 수 있게 연결한다.

## 회원 기본조회 화면 설계 원칙

- 이 화면의 목적은 회원을 빠르게 식별하고 운영상 즉시 확인해야 하는 상태를 한 번에 판단하는 것이다.
- `연락처`, `가입정보`, `상태`, `태그`는 별도 탭이 아니라 `회원 요약` 탭 안의 필드 그룹이나 요약 카드로 배치한다.
- 검색은 회원번호, UID, 이름, 이메일, 휴대폰, CI/DI, 지갑주소, 계좌, 주문번호, 입출금 TXID 역추적을 고려한다.
- 회원 요약에는 회원번호, KYC 상태, 계정상태, 거래가능 여부, 입출금 제한 여부, 위험등급, 최근 로그인, 가입일을 우선 노출한다.
- 핵심 플래그는 휴면, 탈퇴예정, 잠금, FDS 주의, 제재, KYC 재이행 필요, 고액·이상거래 여부처럼 운영 판단에 필요한 상태를 표시한다.
- 관련 화면 바로가기는 KYC 상세, 입출금, 주문·체결, 지갑, 상담, 제재·차단, 감사로그로 연결한다.

## 개발 지침

ADM-MEM-001 화면은 이 문서의 화면 목적, 탭 구성, 설계 방식, 권한/감사 기준을 기준으로 개발한다.
설계와 다르게 구현해야 하는 항목은 변경 사유와 대체 정책을 작업 기록에 남긴다.
