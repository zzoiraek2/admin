# ADM-RISK-003 차단 사유코드 관리

- 상단 메뉴: 준법·FDS 관리
- 좌측 1메뉴: 입출금 차단 관리
- 좌측 2메뉴: 차단 사유코드 관리
- 상태: 준비중
- 구 어드민 매핑: 신규 정책 정의

## 화면 목적

차단사유 표준화

## 업무 설명

사유코드, 보고분류, 내부분류, 사용여부, 변경이력

## 탭 구성

- 코드
- 보고매핑
- 사용여부
- 변경이력

## 설계 방식 제안

- 목록, 상세, 처리 이력, 변경 이력을 기준 구조로 둔다.
- 조회 권한과 처리 권한을 분리한다.
- 변경성 액션은 사유 입력과 감사로그를 필수로 둔다.
- 관련 회원, 거래, 입출금, Case, 보고서 화면으로 이동할 수 있게 연결한다.

## 차단 사유코드 산출 기준

- 사유코드는 코인거래소용 가상자산 입출고와 원화거래소용 예치금 입출금을 구분해서 관리한다.
- 각 사유에는 `사건발생일`, `가상자산종목수`, `이용자수` 같은 건 수 산출 기준을 둔다.
- 산출 기준이 사건발생일인 경우 월간 집계에는 숫자 합계가 아니라 발생 일자 목록을 표시할 수 있어야 한다.
- 산출 기준이 가상자산종목수인 경우 입고만 차단, 출고만 차단, 입출고 모두 차단 범위를 구분한다.
- 산출 기준이 이용자수인 경우 고객 단위 원천 로그와 차단 상태 변경 이력을 집계 근거로 삼는다.

## 개발 지침

ADM-RISK-003 화면은 이 문서의 화면 목적, 탭 구성, 설계 방식, 권한/감사 기준을 기준으로 개발한다.
설계와 다르게 구현해야 하는 항목은 변경 사유와 대체 정책을 작업 기록에 남긴다.
