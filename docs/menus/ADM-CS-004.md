# ADM-CS-004 증빙 관리

## 기본 정보

- 메뉴 코드: ADM-CS-004
- 상단 메뉴: 고객지원·서비스
- 좌측 1메뉴: 증빙센터
- 좌측 2메뉴: 증빙 관리
- 상태: 정의됨
- 구 어드민 매핑: 회원관리 회원관리 > 회원관리 > 증빙 서류 제출 

## 화면 목적

고객 제출 증빙의 조회·검토·보관·파기 통합 관리

## 업무 설명

제출자, 증빙유형, 제출일, 관련업무, 검토상태, 보완요청, 보관기간, 파기결과, 연계 Case


## 결재 프로세스

- 검토·보완: 위변조 의심 통과, 필수 증빙 면제, 수동 통과, 반려 확정은 승인 대상이다.
- 보관·파기: 파기 실행, 예외 보존, 보관기간 연장은 승인 완료 후 반영한다.
- 연계업무: 증빙 연결 자체는 업무 맥락 기록이지만, 증빙을 근거로 상태를 변경하는 원 업무의 승인 정책을 따른다.

## 탭 구성

- 제출내역
- 검토·보완
- 보관·파기
- 연계업무
- 감사로그


## 탭별 상세 설계

### 제출내역

- 업무 목적: 고객이 여러 업무에서 제출한 증빙을 통합 목록으로 조회하고 처리 대상인지 판단하는 탭이다.
- 검색 조건: 기간, 문의번호, 회원번호, 카테고리, 처리상태, 제출경로, 증빙유형, 미처리 여부
- 목록 컬럼: 접수/게시일시, 대상 회원, 카테고리, 증빙유형, 제출경로, 상태, SLA, 최근 변경, 담당자
- 상세보기 정보: 문의/콘텐츠 본문, 제출 원문, 첨부 파일, 관련 회원 맥락, 처리 메모, 연계 업무 링크
- 주요 기능: 목록 조회, 상세 열기, 담당자 배정, 관련 회원/업무 이동, 보완 요청 초안 작성
- 결재/승인: 해당사항 없음
- 연계 메뉴: 회원 기본조회, KYC 심사 대기, 로그인/보안상태, 입출금 처리, 상담 이력, 감사로그
- 권한/감사: 증빙 조회, 열람, 다운로드, 보완 요청, 반려/승인, 파기, 연계 업무 이동을 감사로그로 남긴다.

### 검토·보완

- 업무 목적: 제출 증빙의 진위, 완전성, 업무 적합성을 검토하고 보완 요청이나 반려 판단을 남기는 탭이다.
- 검색 조건: 증빙유형, 검토상태, 위변조 의심, 보완 필요, 담당자, 관련 업무, 위험등급
- 목록 컬럼: 증빙유형, 검토상태, 위변조 의심, 보완항목, 담당자, 보완기한, 처리결과
- 상세보기 정보: 증빙 원본/마스킹본, 자동 검증 결과, 수동 검토 의견, 보완 요청 이력, 반려 사유
- 주요 기능: 자료 열람, 보완 요청, 검토 의견 등록, 수동 판정, 반려/승인 요청, 관련 Case 이동
- 결재/승인: 위변조 의심 통과, 필수 증빙 면제, 수동 통과, 반려 확정은 승인 대상이다.
- 연계 메뉴: 회원 기본조회, KYC 심사 대기, 로그인/보안상태, 입출금 처리, 상담 이력, 감사로그
- 권한/감사: 증빙 조회, 열람, 다운로드, 보완 요청, 반려/승인, 파기, 연계 업무 이동을 감사로그로 남긴다.

### 보관·파기

- 업무 목적: 증빙 보관기간, 만료, 파기 예정 및 파기 결과를 개인정보 보존 기준에 맞춰 관리하는 탭이다.
- 검색 조건: 보관상태, 만료일, 파기예정일, 파기결과, 업무유형, 회원번호
- 목록 컬럼: 증빙유형, 관련 업무, 보관상태, 만료일, 파기예정일, 파기결과, 승인상태
- 상세보기 정보: 보관 근거, 만료 산정 기준, 파기 전후 상태, 파기 승인 이력, 예외 보존 사유
- 주요 기능: 만료 대상 조회, 파기 대상 확인, 파기 상신, 예외 보존 요청, 파기 결과 확인
- 결재/승인: 파기 실행, 예외 보존, 보관기간 연장은 승인 완료 후 반영한다.
- 연계 메뉴: 회원 기본조회, KYC 심사 대기, 로그인/보안상태, 입출금 처리, 상담 이력, 감사로그
- 권한/감사: 증빙 조회, 열람, 다운로드, 보완 요청, 반려/승인, 파기, 연계 업무 이동을 감사로그로 남긴다.

### 연계업무

- 업무 목적: 증빙이 연결된 KYC, 입출금, OTP 해제, 상담, 법정 요청 등 원 업무로 이동해 후속 처리를 확인하는 탭이다.
- 검색 조건: 업무유형, Case 번호, 회원번호, 처리상태, 담당자, 연결일시
- 목록 컬럼: 업무유형, Case 번호, 대상 회원, 처리상태, 증빙 사용목적, 담당자, 최근 변경
- 상세보기 정보: 연계 업무 요약, 증빙 사용 목적, 관련 승인 건, 상담/통지 이력, 다음 처리 링크
- 주요 기능: 관련 업무 이동, 연결 해제 요청, 증빙 사용 이력 확인, 후속 Case 생성
- 결재/승인: 증빙 연결 자체는 업무 맥락 기록이지만, 증빙을 근거로 상태를 변경하는 원 업무의 승인 정책을 따른다.
- 연계 메뉴: 회원 기본조회, KYC 심사 대기, 로그인/보안상태, 입출금 처리, 상담 이력, 감사로그
- 권한/감사: 증빙 조회, 열람, 다운로드, 보완 요청, 반려/승인, 파기, 연계 업무 이동을 감사로그로 남긴다.

### 감사로그

- 업무 목적: 증빙 조회, 열람, 마스킹 해제, 다운로드, 보완 요청, 파기 이력을 감사 기준으로 추적하는 탭이다.
- 검색 조건: 기간, 관리자ID, 작업유형, 회원번호, 증빙유형, IP, 결과
- 목록 컬럼: 작업일시, 관리자, 작업유형, 대상 증빙, 사유, 변경 전후 값, IP, 결과
- 상세보기 정보: 조회 사유, 마스킹 해제 사유, 다운로드 사유, 파기 승인 의견, 관련 업무 이동 이력
- 주요 기능: 감사 로그 조회, 변경 전후 비교, 로그 다운로드, 관련 승인 건 이동
- 결재/승인: 해당사항 없음
- 연계 메뉴: 회원 기본조회, KYC 심사 대기, 로그인/보안상태, 입출금 처리, 상담 이력, 감사로그
- 권한/감사: 증빙 조회, 열람, 다운로드, 보완 요청, 반려/승인, 파기, 연계 업무 이동을 감사로그로 남긴다.

## 설계 방식 제안

- 목록, 상세, 처리 이력, 변경 이력을 기준 구조로 둔다.
- 조회 권한과 처리 권한을 분리한다.
- 변경성 액션은 사유 입력과 감사로그를 필수로 둔다.
- 관련 회원, 거래, 입출금, Case, 보고서 화면으로 이동할 수 있게 연결한다.

## 개발 지침

ADM-CS-004 화면은 이 문서의 화면 목적, 탭 구성, 설계 방식, 권한/감사 기준을 기준으로 개발한다.
설계와 다르게 구현해야 하는 항목은 변경 사유와 대체 정책을 작업 기록에 남긴다.
