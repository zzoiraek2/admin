# ADM-WAL-016 자산 이전 관리

## 기본 정보

- 메뉴 코드: ADM-WAL-016
- 상단 메뉴: 입출금·지갑
- 좌측 1메뉴: 지갑 관리
- 좌측 2메뉴: 자산 이전 관리
- 상태: 준비중
- 구 어드민 매핑: 신규 정책 정의

## 화면 목적

내부 지갑 간 자산 이전 요청, 승인, 서명, 브로드캐스트, 완료 확인을 관리한다.

## 업무 설명

핫/콜드/웜/고객/회사 지갑 간 자산 이동을 요청하고 승인 후 실행하는 처리 화면이다. 실제 자산 이동과 외부 TX가 발생할 수 있으므로 이전 요청 승인, 서명, 브로드캐스트, 취소, 재시도는 결재와 감사로그를 필수로 한다.

## 결재 프로세스

- 이전 요청: 이전 자산, 수량, 출발/도착 지갑, 사유 확정은 결재 상신 대상으로 관리한다.
- 승인·서명: 자산 이전 승인, 서명 요청, 실행 승인은 승인 완료 후 진행한다.
- 브로드캐스트·완료: 브로드캐스트, 재시도, 취소, 대체 처리는 승인 완료 후 실행한다.

## 탭 구성

- 이전 요청
- 이전 검토
- 승인·서명
- 브로드캐스트·완료
- 실패·취소 이력

## 탭별 상세 설계

### 이전 요청

- 업무 목적: 내부 지갑 간 자산 이전 요청을 등록하고 조회한다.
- 검색 조건: 기간, 자산, 출발지갑, 도착지갑, 요청상태, 담당자
- 목록 컬럼: 요청번호, 요청시각, 자산, 수량, 출발지갑, 도착지갑, 이전사유, 요청상태, 담당자
- 상세보기 정보: 이전 목적, 출발/도착 주소, 수량, 수수료 예상, 대사 영향, 요청 사유
- 주요 기능: 이전 요청 등록, 임시저장, 요청 취소, 검토 이동
- 결재/승인: 이전 자산, 수량, 출발/도착 지갑, 사유 확정은 결재 상신 대상으로 관리한다.
- 연계 메뉴: ADM-WAL-013 지갑/주소 관리, ADM-WAL-017 지갑잔고 대사
- 권한/감사: 요청 등록, 수정, 취소 이력을 감사로그로 남긴다.

### 이전 검토

- 업무 목적: 이전 요청의 잔고, 주소, 보안, 대사 영향을 검토한다.
- 검색 조건: 요청번호, 자산, 검토상태, 위험여부, 담당자
- 목록 컬럼: 요청번호, 자산, 수량, 출발잔고, 도착지갑, 검토상태, 위험여부, 담당자
- 상세보기 정보: 출발지갑 잔고, 도착지갑 상태, 지갑 구분, 웜월렛 여부, 대사 영향, 검토 의견
- 주요 기능: 잔고 검증, 주소 검증, 위험 확인, 검토 의견 등록, 결재 상신
- 결재/승인: 해당사항 없음
- 연계 메뉴: ADM-WAL-013 지갑/주소 관리, ADM-WAL-017 지갑잔고 대사, ADM-WAL-018 고객자산 대사
- 권한/감사: 검토 의견, 검증 결과, 상신 이력을 감사로그로 남긴다.

### 승인·서명

- 업무 목적: 승인된 이전 요청에 대해 서명 요청과 실행 승인을 관리한다.
- 검색 조건: 기간, 승인상태, 서명상태, 자산, 요청자
- 목록 컬럼: 상신번호, 요청번호, 자산, 수량, 승인상태, 서명상태, 승인자, 서명자, 실행상태
- 상세보기 정보: 승인 의견, 서명 요청값, 서명 결과, 변경 전후 잔고 예상, 보안 확인
- 주요 기능: 승인, 반려, 보완 요청, 서명 요청, 서명 결과 확인
- 결재/승인: 자산 이전 승인, 서명 요청, 실행 승인은 승인 완료 후 진행한다.
- 연계 메뉴: ADM-SEC-007 감사로그, ADM-WAL-013 지갑/주소 관리
- 권한/감사: 승인/반려, 서명 요청, 서명 결과를 감사로그로 남긴다.

### 브로드캐스트·완료

- 업무 목적: 승인/서명 완료 건의 브로드캐스트와 완료 확인을 관리한다.
- 검색 조건: 기간, 실행상태, 자산, TXID, 실패여부
- 목록 컬럼: 실행시각, 요청번호, 자산, 수량, TXID, 브로드캐스트상태, 컨펌수, 완료상태, 실패사유
- 상세보기 정보: 브로드캐스트 요청값, TXID, 컨펌 진행, 완료 확인, 잔고 반영, 대사 결과
- 주요 기능: 브로드캐스트 실행, 완료 확인, 재시도 상신, TX 확인
- 결재/승인: 브로드캐스트, 재시도, 취소, 대체 처리는 승인 완료 후 실행한다.
- 연계 메뉴: ADM-WAL-017 지갑잔고 대사, ADM-WAL-002 입출금 원장조회
- 권한/감사: 브로드캐스트, TXID, 완료 결과, 실패 사유를 감사로그로 남긴다.

### 실패·취소 이력

- 업무 목적: 자산 이전 실패, 취소, 재시도 이력을 조회한다.
- 검색 조건: 기간, 요청번호, 처리결과, 실패사유, 담당자
- 목록 컬럼: 처리시각, 요청번호, 자산, 처리유형, 처리결과, 실패사유, 담당자, 승인번호
- 상세보기 정보: 실패 원인, 취소 사유, 재시도 결과, 승인 이력, 관련 대사 결과
- 주요 기능: 실패/취소 이력 조회, 재시도 건 이동, 감사로그 이동, 다운로드
- 결재/승인: 해당사항 없음
- 연계 메뉴: ADM-SEC-007 감사로그, ADM-WAL-017 지갑잔고 대사
- 권한/감사: 실패/취소 이력 조회와 다운로드 이력을 감사로그로 남긴다.

## 설계 방식 제안

- 자산 이전은 실제 자산 이동 업무이므로 요청, 검토, 승인, 서명, 브로드캐스트, 완료 확인 단계를 분리한다.
- 재시도, 취소, 대체 처리도 결재와 사유를 필수로 둔다.
- 지갑잔고 대사와 고객자산 대사로 이동해 이전 전후 차이를 확인할 수 있게 한다.

## 개발 지침

ADM-WAL-016 화면은 내부 지갑 간 자산 이전의 결재 기반 실행 흐름을 기준으로 개발한다. 승인 없는 서명, 브로드캐스트, 재시도, 취소는 허용하지 않는다.
설계와 다르게 구현해야 하는 항목은 변경 사유와 대체 정책을 작업 기록에 남긴다.
