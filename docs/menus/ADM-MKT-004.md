# ADM-MKT-004 거래 현황

## 기본 정보

- 메뉴 코드: ADM-MKT-004
- 상단 메뉴: 거래·마켓 관리
- 좌측 1메뉴: 거래 현황
- 좌측 2메뉴: 거래 현황
- 상태: 정의됨
- 구 어드민 매핑: 모니터링 > 거래 현황

## 화면 목적

전체 거래 상태 모니터링

## 업무 설명

거래대금, 체결건수, 주문건수, 취소율, 활성회원, 페어별 시장경보 알림 발생 현황, 기간 기준 시장감시 사건 모니터링

## 탭 구성

- 실시간
- 일별
- 마켓별
- 회원별
- 시장감시 모니터링

## 설계 방식 제안

- 목록, 상세, 처리 이력, 변경 이력을 기준 구조로 둔다.
- 조회 권한과 처리 권한을 분리한다.
- 변경성 액션은 사유 입력과 감사로그를 필수로 둔다.
- 관련 회원, 거래, 입출금, Case, 보고서 화면으로 이동할 수 있게 연결한다.

## 시장감시 모니터링 탭 설계

- 페어별 시장경보 알림 발생 현황을 기간 기준으로 조회할 수 있게 한다.
- 경보 유형, 심각도, 발생 횟수, 최초 발생 시각, 최근 발생 시각, 처리 상태를 주요 컬럼으로 둔다.
- 특정 페어에서 반복 발생하는 경보나 이상 거래 징후를 빠르게 확인할 수 있게 정렬과 필터를 제공한다.
- 기간 검색은 당일, 7일, 30일, 사용자 지정 기간을 기본으로 둔다.
- 경보 상세에서는 관련 주문, 체결, 회원, 알림 로그, 처리 이력을 연결한다.
- 시장경보 알림은 모니터링 목적의 조회 기능으로 우선 설계하고, 조치가 필요한 경우 별도 Case 또는 처리 화면으로 이동시킨다.

## 개발 지침

ADM-MKT-004 화면은 이 문서의 화면 목적, 탭 구성, 설계 방식, 권한/감사 기준을 기준으로 개발한다.
설계와 다르게 구현해야 하는 항목은 변경 사유와 대체 정책을 작업 기록에 남긴다.
