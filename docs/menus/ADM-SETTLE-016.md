## 결재 프로세스

- 대상 요약: 조회와 필터링은 결재 없이 가능하지만 상태 변경이나 다운로드는 권한과 사유를 확인한다.
- 지급·회수 요청: 접수 확인은 담당자 권한으로 가능하되 반려, 예외 접수, 대량 등록은 사유와 승인 로그를 남긴다.
- 승인·실행: 승인 완료 전에는 외부 영향이 있는 상태 변경, 차단, 지급, 게시, 보고값 확정을 실행하지 않는다.
- 실패·보정: 조회와 맥락 확인은 결재 없이 가능하지만 상태 변경이나 외부 영향 조치는 승인 흐름으로 전환한다.
