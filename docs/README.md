# Admin Policy Documents

이 폴더는 어드민 설계정책서를 Git에서 관리하기 위한 문서 저장소입니다.

## 운영 원칙

- `docs/menus/{MENU_CODE}.md`를 메뉴별 설계정책의 원천 문서로 둔다.
- 메뉴 상태도 각 MD 문서의 `- 상태:` 값을 기준으로 한다.
- 웹 UI에서 상태를 변경하면 로컬 관리 서버가 해당 MD 파일을 직접 수정한다.
- 변경된 MD 파일을 커밋하고 푸시하면 다른 사람도 동일한 상태와 정책을 확인한다.
- 여러 메뉴가 하나의 업무 흐름으로 묶이는 경우에는 `docs/domains/`에 도메인 단위 정책 문서를 추가한다.

## 메뉴 문서 위치

- 메뉴별 문서: `docs/menus/`
- 도메인 문서: `docs/domains/`

## 탭 구성 정리 규칙

- 탭은 필드 그룹이 아니라 업무 관점으로 정의한다.
- `기본정보`, `연락처`, `가입정보` 같은 항목은 탭이 아니라 상세 화면 내부의 섹션으로 둔다.
- 조회 화면은 요약, 관련 업무 현황, 리스크, 처리이력, 감사로그 관점으로 나눈다.
- 승인/처리 화면은 요청, 검토, 승인·반려, 적용 결과, 감사로그 관점으로 나눈다.
- 모니터링 화면은 실시간, 기간별, 대상별, 알림·이벤트, 처리현황 관점으로 나눈다.
- 전체 메뉴의 탭 구성은 `scripts/rewrite-menu-tabs.js` 기준으로 정렬하며, 변경 후 `docs/menus-index.json`을 재생성한다.
- 탭별 세부 개발 기준은 `## 탭별 상세 설계`에 기록한다. 각 탭은 업무 목적, 검색 조건, 목록 컬럼, 상세보기 정보, 주요 기능, 결재/승인, 연계 메뉴, 권한/감사 기준을 포함한다.
- 탭별 상세 설계 초안은 `scripts/generate-tab-details.js`로 생성하고, 운영 정책이 확정되는 메뉴는 해당 MD에서 직접 보강한다.

## 개발 지시 예시

`ADM-RISK-004 차단 관리대장 화면은 docs/menus/ADM-RISK-004.md 설계 기준대로 개발해.`

이 지시를 받은 개발자는 해당 문서의 화면 목적, 탭 구성, 집계 기준, 권한/감사 기준을 우선 확인하고 구현해야 합니다.
