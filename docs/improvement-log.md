# Admin Policy Workspace Improvement Log

이 문서는 내부망 패치나 운영 반영 지시 시 기준으로 삼는 개선 이력입니다.
최신 항목을 위에 추가하며, 각 항목은 관련 메뉴, 변경 요약, 패치 기준, 관련 커밋을 함께 기록합니다.

## 2026-06-23

### 전체 메뉴 결재 프로세스 정리 기준 반영

- 전체 메뉴 MD의 `## 결재 프로세스`를 다시 정리해 조회, 필터링, 요약, 현황, 목록, 상세, 모니터링, 이력, 로그, 다운로드, 마스킹 해제 같은 권한/감사 성격의 문구를 결재 대상에서 제거했습니다.
- `조회와 필터링은 결재 없이 가능하지만 상태 변경이나 다운로드는 권한과 사유를 확인한다.` 유형의 문구는 결재 프로세스에서 제거하고, 각 탭의 `결재/승인`은 `해당사항 없음`으로 정리했습니다.
- 상단 결재 프로세스에는 상태 변경, 정책값 변경, 고객 잔고 반영, 외부 전송, 차단/해제, 예외 등록, 게시, 보고값 확정, 자산 이전 실행처럼 실제 승인 필요한 행위만 남기도록 기준을 반영했습니다.
- `운영정책 요약`, `대상 요약`, `적용 이력`, `감사로그`처럼 조회 또는 기록 확인 목적의 탭은 결재 프로세스에 노출되지 않도록 정리했습니다.
- 결재 대상 행위가 탭명과 맞지 않는 일부 메뉴는 탭명 또는 결재 문구를 보정했습니다. 예: `ADM-RISK-003`의 `사유코드 목록`을 `사유코드 등록·변경`으로, `ADM-LIST-004`의 `프로젝트 현황`을 `프로젝트 정보 검토`로 정리했습니다.
- 이번 패치 기준: 전체 `docs/menus/*.md` 결재 프로세스 정규화, `menus-index.json`

### ADM-MEM-013 고액자산가 조회 역할 재정리

- `ADM-MEM-013 고액자산가`를 고액자산 보유 고객의 위험관리 화면이 아니라 기준시점별 고객 자산 평가액과 고액자산가 해당 여부를 조회하는 화면으로 재정의했습니다.
- 탭은 `고액자산가 요약`, `고객별 자산 상세`, `평가·산출 기준`, `변동 이력`으로 정리했습니다.
- `신청·증빙`, `검토·보완`, `승인·반려`, `제한·후속조치` 탭은 업무 성격에 맞지 않아 제거했습니다.
- AML 위험평가, 고객 위험등급 산정, EDD/STR 판단은 AML 시스템 역할로 명시하고 어드민은 AML로 제공되는 자산 평가 기준 데이터와 산출 근거 확인에 한정했습니다.
- AML 전송 현황, 전송 실패, 재전송 관리는 API 실시간 연계 구조이므로 별도 탭이나 업무 기능으로 두지 않도록 정리했습니다.
- 결재 프로세스는 조회 화면 성격에 맞춰 `해당사항 없음`으로 고정했습니다.
- 이번 패치 기준: `ADM-MEM-013.md`, `menus-index.json`
