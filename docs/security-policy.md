# Admin Security Policy

이 문서는 전체 어드민 메뉴에 공통으로 적용하는 보안설계 정책입니다. 메뉴별 설계정책서(`docs/menus/*.md`)가 별도 예외를 명시하지 않는 한, 모든 조회, 처리, 다운로드, 원문 열람, 상태 변경 기능은 본 문서 기준을 우선 적용합니다.

## 1. 적용 원칙

- 어드민 화면은 최소조회, 최소노출, 최소권한 원칙으로 설계한다.
- 개인정보, 거래, 입출금, 자산, 로그, 증빙 자료는 기본 목록 자동조회를 금지한다.
- 목록형 화면은 검색조건 또는 기간조건을 지정한 뒤 조회하도록 한다.
- 개인정보는 목록과 상세 모두 기본 마스킹 표시를 원칙으로 한다.
- 원문 열람, 마스킹 해제, 다운로드, 대량조회, 대량처리, 정책값 변경은 사유와 감사로그를 남긴다.
- 결재 프로세스와 보안 통제는 구분한다. 조회, 필터링, 마스킹, 다운로드 사유 기록은 결재가 아니라 권한/감사 통제이다.
- 법령, 내부 규정, ISMS-P 심사 기준, 전자금융/가상자산 업무 기준이 더 엄격한 경우 더 엄격한 기준을 따른다.

## 2. 검색/조회 통제

### 2.1 기본 자동조회 금지

다음 성격의 화면은 초기 진입 시 데이터를 자동 조회하지 않는다.

- 회원, KYC, 인증, 로그인/보안, 고객지원, 증빙, 민원
- 주문, 체결, 입출금, 지갑, 자산, 정산, 회계
- FDS, KYT, 차단, 제재, 법정요청, 수사협조
- 관리자 감사로그, 접속기록, 권한 이력

### 2.2 필수 검색조건

목록형 화면은 업무 성격에 맞춰 하나 이상의 필수 조건을 요구한다.

- 회원 단위: 회원번호, CI, 휴대폰, 이메일, 이름, 고객 식별번호 중 1개 이상
- 거래/체결/입출금 단위: 기간, 회원번호, 주문번호, 거래번호, TXID, 자산, 마켓 중 필요한 조합
- 로그 단위: 기간, 관리자 ID, 메뉴, 행위유형, 대상 식별값 중 필요한 조합
- 증빙/민원 단위: 기간, 접수번호, 회원번호, 처리상태, 담당자 중 필요한 조합

### 2.3 기간 및 건수 제한

- 일반 목록 조회 기간은 기본 1개월 이하로 제한한다.
- 대량 데이터 화면은 메뉴별로 최대 조회 기간과 최대 조회 건수를 정의한다.
- 기준을 초과하는 조회는 조건 축소 안내를 표시하고 실행하지 않는다.
- 운영상 대량조회가 필요한 경우 별도 권한과 사유 입력을 요구한다.

### 2.4 다운로드 통제

- 다운로드는 별도 탭으로 분리하지 않고 목록 기능으로 제공하되, 권한과 사유를 필수 확인한다.
- 다운로드 전 조회조건, 예상 건수, 포함 개인정보 항목을 표시한다.
- 다운로드 파일에는 생성자, 생성시각, 메뉴코드, 추적번호를 포함한다.
- 다운로드 이력은 관리자 ID, 메뉴, 조회조건, 건수, 사유, 파일명, IP, 시각을 감사로그에 기록한다.

## 3. 개인정보 마스킹 정책

### 3.1 기본 기준

- 목록 화면은 마스킹 값을 기본 표시한다.
- 상세 화면도 업무상 원문 확인이 필요한 항목을 제외하고 마스킹 표시한다.
- 원문 열람은 권한, 사유, 대상, 열람 시각을 감사로그에 남긴다.
- 주민등록번호, 외국인등록번호, 여권번호, 운전면허번호, 계좌번호, 카드번호, CI는 기본 원문 노출을 금지한다.

### 3.3 원문 열람

원문 열람 또는 마스킹 해제는 다음 조건을 모두 만족해야 한다.

- 원문 열람 권한 보유
- 열람 사유 선택 또는 직접 입력
- 대상 회원, 대상 항목, 메뉴코드, 처리 Case 연결
- 열람 후 감사로그 자동 기록

원문 열람 감사로그에는 관리자 ID, 부서, 메뉴, 대상 식별값, 항목명, 사유, 열람 시각, 접속 IP, 세션 ID를 기록한다.

## 4. 계정, 인증, 세션 정책

### 4.1 관리자 계정

- 관리자 계정은 개인별 계정으로 발급하고 공용 계정을 금지한다.
- 계정 생성, 권한 부여, 권한 변경, 권한 말소는 승인된 요청에 근거해야 한다.
- 퇴사, 휴직, 부서이동, 직무변경 시 권한 회수 또는 재검토를 수행한다.
- 장기 미사용 계정은 잠금 또는 비활성화한다.

### 4.2 비밀번호 및 잠김

- 비밀번호 복잡도, 최소 길이, 변경 주기, 재사용 제한을 시스템 설정으로 관리한다.
- 로그인 실패 임계치를 초과하면 계정을 잠근다.
- 잠금 해제는 본인확인 또는 관리자 승인 절차를 거친다.
- 초기 비밀번호 또는 임시 비밀번호는 최초 로그인 시 변경하도록 한다.

### 4.3 세션 통제

- 동일 관리자 계정의 동시 세션을 금지한다.
- 일정 시간 미사용 시 자동 로그아웃한다.
- 중요 기능 접근, 원문 열람, 다운로드, 정책 변경, 승인 처리 시 세션 유효성을 재확인한다.
- 세션 탈취 방지를 위해 세션 ID 재사용과 장시간 고정 세션을 제한한다.

## 5. 권한 설계

- 메뉴 접근권한, 조회권한, 원문 열람권한, 다운로드권한, 등록/수정/삭제권한, 승인권한을 분리한다.
- 동일 사용자가 요청자와 승인자를 동시에 수행하지 못하도록 역할 충돌을 제한한다.
- 민감 메뉴는 업무 역할과 조직 기준으로 접근 범위를 제한한다.
- 권한 변경은 변경 전/후 값, 요청자, 승인자, 사유, 적용 시각을 기록한다.
- 권한 매트릭스는 정기 점검 대상이며, 불필요 권한은 회수한다.

## 6. 감사로그 정책

### 6.1 필수 기록 대상

다음 행위는 감사로그를 남긴다.

- 관리자 로그인 및 로그아웃
- 로그인 실패, 계정 잠김, 잠금 해제
- 사용자 권한 부여, 변경, 말소
- 정보시스템 시작, 중지, 배포, 설정 변경
- 메뉴 접근, 목록 조회, 상세 조회
- 개인정보 원문 열람 및 마스킹 해제
- 다운로드, 출력, 파일 생성
- 등록, 수정, 삭제, 승인, 반려, 상태 변경
- 대량조회, 대량처리, 외부 전송
- 결재 상신, 승인, 반려, 회수, 재상신

### 6.2 필수 기록 항목

- 관리자 ID, 이름, 부서, 역할
- 메뉴코드, 화면명, 기능명, 행위유형
- 대상 식별값, 변경 전 값, 변경 후 값
- 조회조건, 조회건수, 다운로드건수
- 사유, 승인번호, Case 번호
- 접속 IP, 브라우저/클라이언트 정보, 세션 ID
- 요청 시각, 처리 결과, 실패 사유

### 6.3 보존 및 위변조 방지

- 감사로그는 운영자가 임의 수정하거나 삭제할 수 없도록 한다.
- 로그 저장소 접근권한은 운영권한과 분리한다.
- 로그는 검색 가능해야 하며, ISMS-P 점검 시 샘플 추출이 가능해야 한다.
- 로그 보존기간은 내부 규정과 관련 법령 기준을 따른다.

## 7. 증빙, 파일, 보존기간, 유효기간 관리

### 7.1 관리 대상

다음 자료는 보존기간과 유효기간 관리 대상이다.

- KYC, KYB, CARF, 고객확인, 본인확인 증빙
- 민원, 상담, 신고, 법정요청, 수사협조 문서
- 입출금, 오입금, 반환, 제한, 차단, 해제 관련 첨부파일
- 거래지원 심사자료, 프로젝트 제출자료, 의결자료
- 결재 첨부파일, 승인 근거, 운영 변경 신청서
- 다운로드 파일, 대량처리 결과 파일, 외부 전송 결과 파일

### 7.2 보존기간 설정

- 파일 유형별 보존기간을 설정값으로 관리한다.
- 보존기간은 문서 유형, 업무 도메인, 법정 보존 필요성, 분쟁 대응 필요성을 기준으로 구분한다.
- 보존기간 만료 전 알림 대상을 생성한다.
- 만료 자료는 파기, 별도 보관, 보존 연장 중 하나로 처리한다.
- 보존 연장은 사유와 승인 근거를 기록한다.

### 7.3 유효기간 설정

다음 자료는 유효기간을 관리한다.

- 신분증, 사업자등록증, 법인등기, 위임장
- 법원, 수사기관, 감독기관 공문
- 제재, 차단, 동결, 반환, 상속, 법정대리 관련 증빙
- 프로젝트 심사자료, 기술검토 자료, 외부 평가자료

유효기간이 만료된 증빙은 신규 처리의 근거로 사용할 수 없으며, 재제출 또는 갱신 요청 대상으로 표시한다.

### 7.4 파기 및 예외

- 파기 대상은 파기 전 목록 검토를 수행한다.
- 파기 결과는 파일 식별값, 대상 건수, 파기 방식, 담당자, 시각, 결과로 기록한다.
- 소송, 수사, 분쟁, 감독기관 요청 등 보존 예외 사유가 있으면 파기를 보류한다.
- 파기 보류와 보존 연장은 승인 근거와 만료 예정일을 함께 기록한다.

## 8. 화면 개발 공통 체크리스트

메뉴 개발 시 다음 항목을 확인한다.

- 초기 진입 시 자동조회가 필요한 화면인지, 아니라면 조회 전 조건 입력을 강제했는가
- 필수 검색조건, 조회 기간 상한, 최대 조회 건수가 정의되어 있는가
- 목록과 상세의 개인정보 항목이 기본 마스킹되는가
- 원문 열람이 필요한 항목은 권한, 사유, 감사로그가 연결되어 있는가
- 다운로드 기능은 권한, 사유, 건수, 파일 추적값, 감사로그를 남기는가
- 등록, 수정, 삭제, 상태 변경, 승인, 반려 행위가 변경 전/후 값과 함께 기록되는가
- 결재가 필요한 행위와 단순 권한/감사 통제가 구분되어 있는가
- 증빙 파일은 보존기간과 유효기간 관리 대상인지 정의되어 있는가
- 관리자 권한과 업무 역할이 분리되어 있는가
- ISMS-P 점검 시 제출 가능한 로그와 설정 증적이 남는가

## 9. ISMS-P 실사 확인 증적

실사 대응 시 다음 증적을 준비할 수 있어야 한다.

- 관리자 계정 목록과 권한 매트릭스
- 권한 부여, 변경, 말소 이력
- 로그인, 로그아웃, 로그인 실패, 계정 잠김 이력
- 비밀번호 정책과 잠김 임계치 설정 화면
- 동일 사용자 동시 세션 제한 설정 또는 테스트 결과
- 개인정보 마스킹 화면 캡처
- 원문 열람 사유 및 감사로그 샘플
- 다운로드 이력과 파일 추적번호 샘플
- 중요 설정 변경 이력과 승인 근거
- 증빙 파일 보존기간 설정표
- 유효기간 만료 증빙 목록과 재제출/갱신 처리 이력
- 보존기간 만료 파일 파기 이력과 보존 예외 승인 이력
- 정보시스템 시작, 중지, 배포, 설정 변경 로그
